## [3.5.2] — 2026-04-05

### ✨ New Features

-**Рідна інтеграція Qoder API:**Повністю оновлено Qoder Executor, щоб обійти застарілий алгоритм шифрування COZY AES/RSA, направляючи безпосередньо на рідну URL-адресу, сумісну з DashScope OpenAi. Усуває складні залежності від модулів Node `crypto`, одночасно покращуючи точність потоку.
—**Капітальний ремонт механізму стійкості:**інтегроване витончене резервне копіювання переповнення контексту, проактивне виявлення маркерів OAuth і запобігання викиду порожнього вмісту (#990).
—**Контекстно-оптимізована стратегія маршрутизації:**додано нову можливість інтелектуальної маршрутизації для власного збільшення вікон контексту в автоматизованих комбінованих розгортаннях (#990).### 🐛 Bug Fixes

-**Пошкодження потоку API відповідей:**Виправлено пошкодження глибокого клонування, коли межі перекладу Anthropic/OpenAI видаляли `response.` специфічні префікси SSE з меж потоку (#992). -**Claude Cache Passthrough Alignment:**вирівняні CC-сумісні маркери кешу узгоджено з режимом Client Pass-Through угорі, зберігаючи швидке кешування.
—**Витік пам’яті Turbopack:**прикріплено Next.js до суворої версії 16.0.10, що запобігає витоку пам’яті та застарілості збірки через нещодавні хешовані регресії модуля Turbopack (#987).---

### 🔧 Hotfix Patch — Critical Bug Fixes

Три критичні регресії, про які повідомляли користувачі після запуску версії 3.0.0, усунено.#### fix(translator): strip `proxy_` prefix in non-streaming Claude responses (#605)

Префікс `proxy_`, доданий Клодом OAuth, було видалено лише з**потокових**відповідей. У режимі**non-streaming**`translateNonStreamingResponse` не мав доступу до `toolNameMap`, через що клієнти отримували спотворені назви інструментів, наприклад `proxy_read_file` замість `read_file`.

LongCat AI не надає `GET /v1/models`. Загальний валідатор `validateOpenAICompatibleProvider` перейшов до резервного варіанту завершення чату, лише якщо було встановлено `validationModelId`, який LongCat не налаштовує. Це призвело до збою перевірки постачальника з оманливою помилкою під час додавання/збереження.

**Виправлення:**додано `longcat` до карти спеціальних валідаторів, безпосередньо перевіряючи `/chat/completions` і розглядаючи будь-яку неавтентифіковану відповідь як пропуск.#### fix(translator): normalize object tool schemas for Anthropic (#595)

Інструменти MCP (наприклад, `pencil`, `computer_use`) пересилають визначення інструментів за допомогою `{type:"object"}`, але без поля `properties`. API Anthropic відхиляє їх за допомогою: `схеми об’єкта відсутні властивості`.

**Виправлення:**у `openai-to-claude.ts` вставте `properties: {}` як безпечне значення за умовчанням, коли `type` є `"object"`, а `properties` відсутній.---
