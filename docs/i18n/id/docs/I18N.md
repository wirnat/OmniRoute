### Source of Truth

-**String UI**: `src/i18n/messages/en.json` (sumber bahasa Inggris, ~2800 kunci) -**File lokal**: `src/i18n/messages/{locale}.json` (30 terjemahan) -**Framework**: `next-intl` dengan resolusi lokal berbasis cookie -**Config**: `src/i18n/config.ts` — mendefinisikan 30 lokal, nama bahasa, bendera### Runtime Flow

1. Pengguna memilih bahasa → kumpulan cookie `NEXT_LOCALE`
2. `src/i18n/request.ts` menyelesaikan lokal: cookie → header `Accept-Language` → fallback `en`
3. Impor dinamis memuat `messages/{locale}.json`
4. Komponen menggunakan `useTranslations("namespace")` dan `t("key")`### Supported Locales

### 1. Register the Locale

Sunting `src/i18n/config.ts`:```ts
// Add to LOCALES array
"xx",
// Add to LANGUAGES array
{ code: "xx", label: "XX", name: "Language Name", flag: "🏳️" },

````

### 2. Add to Generator

Edit `scripts/i18n/generate-multilang.mjs` — tambahkan entri ke `LOCALE_SPECS`:```js
{
  code: "xx",
  googleTl: "xx",
  label: "XX",
  flag: "🏳️",
  languageName: "Language Name",
  readmeName: "Language Name",
  docsName: "Language Name",
},
````

### 3. Generate Initial Translation

```bash
node scripts/i18n/generate-multilang.mjs messages
```

Ini membuat `src/i18n/messages/xx.json` diterjemahkan secara otomatis dari `en.json` melalui Google Terjemahan.### 4. Review & Fix Auto-Translations

Terjemahan otomatis adalah titik awal. Tinjau secara manual untuk:

- Akurasi teknis
- Terminologi yang sesuai konteks
- Penanganan placeholder yang tepat (`{count}`, `{value}`, dll.)### 5. Validate

```bash
python3 scripts/validate_translation.py quick -l xx
python3 scripts/validate_translation.py diff common -l xx
```

### 6. Generate Translated Documentation

```bash
node scripts/i18n/generate-multilang.mjs docs
```

## Auto-Translation Pipeline

### generate-multilang.mjs (Google Translate)

**Mesin terjemahan otomatis utama**— menggunakan API gratis Google Terjemahan untuk menghasilkan terjemahan untuk string UI, README, dan dokumentasi.```bash
node scripts/i18n/generate-multilang.mjs [messages|readme|docs|all]

````

| Modus | Apa fungsinya |
| ---------- | --------------------------------------------------------------- |
| `pesan` | Menerjemahkan kunci yang hilang di `src/i18n/messages/{locale}.json` dari `en.json` |
| `baca saya` | Menerjemahkan `README.md` ke semua lokal sebagai `README.{code}.md` di root proyek |
| `dokumen` | Menerjemahkan `DOC_SOURCE_FILES` menjadi `docs/i18n/{locale}/{docName}` |
| `semua` | Menjalankan ketiga mode |

**Fitur:**

-**Perlindungan teks**: Menyamarkan blok kode (` ``` `), kode sebaris (`` ` ``), tautan/gambar penurunan harga (`[teks](url)`), tag HTML, tabel, dan placeholder ICU (`{count}`, `{value}`, `{total}`, dll.) sebelum penerjemahan, lalu memulihkannya
-**Pengelompokan terpotong**: Menggabungkan beberapa string dengan pembatas `__OMNIROUTE_I18N_SEPARATOR__` untuk meminimalkan panggilan API (maks 1800 karakter per permintaan)
-**Cache dalam memori**: Menghindari panggilan API yang berlebihan untuk string berulang dalam satu sesi
-**Coba ulang logika**: Backoff eksponensial (hingga 5 upaya dengan penundaan upaya 300 md ×) untuk kesalahan 429/5xx
-**Waktu habis**: 20 detik per permintaan
-**Lewati yang ada**: Jika file target sudah ada, maka TIDAK akan ditimpa

**Perilaku penting:**

- `docs/i18n/README.md`**dibuat ulang**setiap kali dijalankan — ini adalah indeks semua dokumen yang dibuat secara otomatis
- File root `README.{code}.md` hanya dibuat jika tidak ada (lewati lokal di `EXISTING_README_CODES`)
- Bilah bahasa (`🌐**Bahasa:**...`) secara otomatis dimasukkan/diperbarui di semua dokumen yang diterjemahkan### i18n_autotranslate.py (LLM-based)

**Penerjemah sekunder**— menggunakan API LLM apa pun yang kompatibel dengan OpenAI (termasuk OmniRoute itu sendiri) untuk menerjemahkan file penurunan harga `docs/i18n/` yang ada. Terbaik untuk memoles atau menerjemahkan ulang dokumen dengan kualitas lebih baik daripada Google Terjemahan.```bash
python3 scripts/i18n_autotranslate.py \
  --api-url http://localhost:20128/v1 \
  --api-key sk-your-key \
  --model gpt-4o
````

**Fitur:**

- Memindai file penurunan harga `docs/i18n/` untuk paragraf bahasa Inggris
- Melewati blok kode, tabel, dan konten yang sudah diterjemahkan
- Mengirim paragraf ke LLM dengan prompt sistem terjemahan teknis
- Mendukung semua 30 bahasa## Validation & QA

### validate_translation.py

**Validator terjemahan**— membandingkan JSON lokal apa pun dengan `en.json` dan melaporkan masalah.```bash

# Quick check (counts only)

python3 scripts/validate_translation.py quick -l cs

# Output:

# Missing: 0

# Untranslated: 0

# Ignored (UNTRANSLATABLE_KEYS): 236

# Detailed diff by category

python3 scripts/validate_translation.py diff common -l cs
python3 scripts/validate_translation.py diff settings -l cs

# Export to CSV

python3 scripts/validate_translation.py csv -l cs > report.csv

# Export to Markdown

python3 scripts/validate_translation.py md -l cs > report.md

# Full report (default)

python3 scripts/validate_translation.py -l cs

````

**Mendeteksi:**

-**Kunci hilang**— kunci di `en.json` tetapi tidak di file lokal
-**Kunci ekstra**— kunci di file lokal tetapi tidak di `en.json`
-**Kunci yang belum diterjemahkan**— kunci yang nilai lokalnya sama dengan sumber bahasa Inggris (tidak termasuk daftar yang diizinkan)
-**Placeholder mismatches**— ICU placeholders that don't match between source and translation

**Kode keluar:**
| Kode | Arti |
|------|---------|
| 0 | oke |
| 1 | Kesalahan umum |
| 2 | String hilang (kesalahan besar) |
| 3 | Peringatan yang tidak diterjemahkan (lembut) |

**Lingkungan:**Setel `TRANSLATION_LANG=cs` atau gunakan tanda `-l cs`.### check_translations.py

**Pemeriksa kunci kode-ke-JSON**— memindai `src/**/*.tsx` dan `src/**/*.ts` untuk panggilan `useTranslations()` dan memverifikasi semua kunci referensi yang ada di `en.json`.```bash
# Basic check
python3 scripts/check_translations.py

# Verbose output
python3 scripts/check_translations.py --verbose

# Auto-fix (adds missing keys to en.json)
python3 scripts/check_translations.py --fix
````

### untranslatable-keys.json

**File:**`scripts/i18n/untranslatable-keys.json`

Daftar kunci yang diizinkan harus tetap identik dengan sumber bahasa Inggris. Digunakan oleh `validate_translation.py` untuk menghindari peringatan positif palsu "tidak diterjemahkan".```json
{
"description": "Keys that should remain untranslated...",
"keys": [
"common.model",
"common.oauth",
"health.cpu",
...
]
}

````

**Apa yang termasuk di sini:**

- Nama merek/produk: `landing.brandName`, `common.social-github`
- Istilah/akronim teknis: `health.cpu`, `mcpDashboard.pid`, `settings.ai`
- String ICU/format: `apiManager.modelsCount`, `health.millisecondsShort`
- Nilai placeholder: `providers.openaiBaseUrlPlaceholder`, `cliTools.baseUrlPlaceholder`
- Nama protokol: `common.http`, `common.oauth`, `providers.oauth2Label`
- Bagian navigasi: `sidebar.primarySection`, `sidebar.cliSection`

**Untuk menambahkan kunci:**Edit array `keys` di `scripts/i18n/untranslatable-keys.json` dan jalankan kembali validasi.## CI Integration

## Best Practices

### When Editing Translations

1.**Selalu edit `en.json` terlebih dahulu**— ini adalah sumber kebenarannya
2.**Jalankan `generate-multilang.mjs messages`**untuk menyebarkan kunci baru ke semua lokal
3.**Tinjau terjemahan otomatis**— Google Terjemahan adalah titik awal, bukan titik akhir
4.**Validasi sebelum melakukan**— `python3 scripts/validate_translation.py quick -l <lang>`
5.**Perbarui `untranslatable-keys.json`**jika kunci harus tetap dalam bahasa Inggris### Placeholder Safety

- Placeholder ICU (`{count}`, `{value}`, `{total}`, `{seconds}`) harus dipertahankan dengan tepat
- Format jamak (`{count, plural, one {# model} other {# models}}`) harus mempertahankan struktur
- Validator mendeteksi ketidakcocokan placeholder secara otomatis### Adding New Translation Keys in Code

```tsx
// Use namespaced keys
const t = useTranslations("settings");
t("cacheSettings"); // maps to settings.cacheSettings in JSON

// Run check_translations.py to verify keys exist
python3 scripts/check_translations.py --verbose
````

### RTL Considerations

- Arab (`ar`) dan Ibrani (`he`) adalah lokal RTL
- Hindari CSS `kiri`/`kanan` yang dikodekan secara hardcode — gunakan properti logika `mulai`/`akhir`
- Visual QA menangkap ketidakcocokan tata letak RTL melalui `run-visual-qa.mjs`## Known Issues & History

### `in.json` → `hi.json` Fix

Generator awalnya menggunakan `kode: "in"` (kode Google Terjemahan tidak berlaku lagi) untuk bahasa Hindi, bukan ISO 639-1 `hi` yang benar. Tindakan ini membuat duplikat `in.json` yang tidak ada lagi dari `hi.json`. Diperbaiki dengan mengubah `code: "in"` menjadi `code: "hi"` di `generate-multilang.mjs` dan menghapus file yatim piatu.### `docs/i18n/README.md` Is Auto-Generated
