### Source of Truth

-**Kullanıcı arayüzü dizeleri**: `src/i18n/messages/en.json` (İngilizce kaynak, ~2800 anahtar) -**Yerel ayar dosyaları**: `src/i18n/messages/{locale}.json` (30 çeviri) -**Çerçeve**: Çerez tabanlı yerel ayar çözümlemeli "next-intl" -**Config**: `src/i18n/config.ts` — 30 yerel ayarın, dil adlarının ve işaretlerin tümünü tanımlar### Runtime Flow

1. Kullanıcı dili seçer → `NEXT_LOCALE` çerez seti
2. `src/i18n/request.ts` yerel ayarı çözer: çerez → `Accept-Language` başlığı → geri dönüş `en`
3. Dinamik içe aktarma `messages/{locale}.json`u yükler
4. Bileşenler `useTranslations("namespace")` ve `t("key")` kullanır### Supported Locales

### 1. Register the Locale

'src/i18n/config.ts'yi düzenleyin:```ts
// Add to LOCALES array
"xx",
// Add to LANGUAGES array
{ code: "xx", label: "XX", name: "Language Name", flag: "🏳️" },

````

### 2. Add to Generator

`scripts/i18n/generate-multilang.mjs`yi düzenleyin — 'LOCALE_SPECS'e giriş ekleyin:```js
{
  code: "xx",
  googleTl: "xx",
  label: "XX",
  flag: "🏳️",
  languageName: "Language Name",
  readmeName: "Language Name",
  docsName: "Language Name",
},
````

### 3. Generate Initial Translation

```bash
node scripts/i18n/generate-multilang.mjs messages
```

Bu, Google Çeviri aracılığıyla "en.json"dan otomatik olarak çevrilen "src/i18n/messages/xx.json" dosyasını oluşturur.### 4. Review & Fix Auto-Translations

Otomatik çeviriler bir başlangıç ​​noktasıdır. Şunlar için manuel olarak inceleyin:

- Teknik doğruluk
- Bağlama uygun terminoloji
- Yer tutucuların doğru şekilde kullanılması (`{count}`, `{value}`, vb.)### 5. Validate

```bash
python3 scripts/validate_translation.py quick -l xx
python3 scripts/validate_translation.py diff common -l xx
```

### 6. Generate Translated Documentation

```bash
node scripts/i18n/generate-multilang.mjs docs
```

## Auto-Translation Pipeline

### generate-multilang.mjs (Google Translate)

**Birincil otomatik çeviri motoru**— Kullanıcı arayüzü dizeleri, README'ler ve belgeler için çeviriler oluşturmak amacıyla Google Translate ücretsiz API'sini kullanır.```bash
node scripts/i18n/generate-multilang.mjs [messages|readme|docs|all]

````

| Modu | Ne işe yarar |
| ---------- | ----------------------------------------------------------------------------- |
| 'mesajlar' | 'src/i18n/messages/{locale}.json' dosyasındaki eksik anahtarları 'en.json'dan çevirir |
| 'beni oku' | `README.md` dosyasını tüm yerel ayarlara proje kökünde `README.{code}.md` olarak çevirir |
| 'dokümanlar' | "DOC_SOURCE_FILES" dosyasını "docs/i18n/{locale}/{docName}" biçimine çevirir |
| 'hepsi' | Her üç modu da çalıştırır |

**Özellikler:**

-**Metin koruması**: Kod bloklarını (` ``` `), satır içi kodu (`` ` ``), işaretleme bağlantılarını/görüntülerini (`[text](url)`), HTML etiketlerini, tabloları ve ICU yer tutucularını (`{count}`, `{value}`, `{total}`, vb.) çeviriden önce maskeler ve ardından bunları geri yükler
-**Yığınlanmış toplu işlem**: API çağrılarını en aza indirmek için birden çok dizeyi `__OMNIROUTE_I18N_SEPARATOR__` sınırlayıcılarla birleştirir (istek başına maksimum 1800 karakter)
-**Bellek içi önbellek**: Bir oturumda tekrarlanan dizeler için gereksiz API çağrılarını önler
-**Yeniden deneme mantığı**: 429/5xx hataları için üstel geri çekilme (300 ms × deneme gecikmesiyle 5 denemeye kadar)
-**Zaman Aşımı**: İstek başına 20 saniye
-**Mevcut olanı atla**: Hedef dosya zaten mevcutsa üzerine YAZILMAZ

**Önemli davranışlar:**

- `docs/i18n/README.md` her çalıştırmada**yeniden oluşturulur**- bu, tüm dokümanların otomatik olarak oluşturulan bir dizinidir
- Kök `README.{code}.md` dosyaları yalnızca mevcut olmadıklarında oluşturulur (`EXISTING_README_CODES` içindeki yerel ayarlar atlanır)
- Dil çubukları (`🌐**Diller:**...`) çevrilmiş tüm dokümanlara otomatik olarak eklenir/güncellenir### i18n_autotranslate.py (LLM-based)

**İkincil çevirmen**— mevcut "docs/i18n/" işaretleme dosyalarını çevirmek için herhangi bir OpenAI uyumlu LLM API'sini (OmniRoute'un kendisi dahil) kullanır. Dokümanları Google Çeviri'den daha iyi kalitede cilalamak veya yeniden çevirmek için en iyisi.```bash
python3 scripts/i18n_autotranslate.py \
  --api-url http://localhost:20128/v1 \
  --api-key sk-your-key \
  --model gpt-4o
````

**Özellikler:**

- İngilizce paragraflar için `docs/i18n/` işaretleme dosyalarını tarar
- Kod bloklarını, tabloları ve önceden çevrilmiş içeriği atlar
- Teknik çeviri sistemi istemiyle paragrafları LLM'ye gönderir
- 30 dilin tamamını destekler## Validation & QA

### validate_translation.py

**Çeviri doğrulayıcı**— herhangi bir yerel JSON'u "en.json" ile karşılaştırır ve sorunları bildirir.```bash

# Quick check (counts only)

python3 scripts/validate_translation.py quick -l cs

# Output:

# Missing: 0

# Untranslated: 0

# Ignored (UNTRANSLATABLE_KEYS): 236

# Detailed diff by category

python3 scripts/validate_translation.py diff common -l cs
python3 scripts/validate_translation.py diff settings -l cs

# Export to CSV

python3 scripts/validate_translation.py csv -l cs > report.csv

# Export to Markdown

python3 scripts/validate_translation.py md -l cs > report.md

# Full report (default)

python3 scripts/validate_translation.py -l cs

````

**Algılar:**

-**Eksik anahtarlar**— anahtarlar `en.json'da ancak yerel ayar dosyasında değil
-**Ekstra anahtarlar**— yerel ayar dosyasındaki anahtarlar ancak `en.json`daki anahtarlar
-**Çevrilmemiş anahtarlar**— yerel değerin İngilizce kaynağa eşit olduğu anahtarlar (izin verilenler listesi hariç)
-**Yer tutucu uyumsuzlukları**— Kaynak ve çeviri arasında eşleşmeyen YBÜ yer tutucuları

**Çıkış kodları:**
| Kod | Anlamı |
|------|------------|
| 0 | tamam |
| 1 | Genel hata |
| 2 | Eksik dizeler (zor hata) |
| 3 | Çevrilmemiş uyarı (yumuşak) |

**Ortam:**'TRANSLATION_LANG=cs'yi ayarlayın veya '-l cs' işaretini kullanın.### check_translations.py

**JSON kodundan anahtar denetleyici**— `useTranslations()` çağrıları için `src/**/*.tsx` ve `src/**/*.ts`yi tarar ve başvurulan tüm anahtarların `en.json`da mevcut olduğunu doğrular.```bash
# Basic check
python3 scripts/check_translations.py

# Verbose output
python3 scripts/check_translations.py --verbose

# Auto-fix (adds missing keys to en.json)
python3 scripts/check_translations.py --fix
````

### generate-qa-checklist.mjs

**Statik analiz QA**— Next.js sayfa dosyalarını i18n risk ölçümleri için tarar ve bir Markdown raporu oluşturur.```bash

````

**Kontroller:**

- Öncelikli yerel ayarlardaki README dil seçme çubukları ('es', 'fr', 'de', 'ja', 'ar')

**Çıktı:**`docs/reports/i18n-qa-checklist-{date}.md`### run-visual-qa.mjs

- Metin taşması

**Çıktı:**`docs/reports/i18n-visual-qa-{date}.md` + JSON raporu## Managing Untranslatable Keys

### untranslatable-keys.json

**Dosya:**`scripts/i18n/untranslatable-keys.json`

İngilizce kaynakla aynı kalması gereken anahtarların izin verilenler listesi. Yanlış pozitif "çevrilmemiş" uyarıları önlemek için `validate_translation.py` tarafından kullanılır.```json
{
"description": "Keys that should remain untranslated...",
"keys": [
"common.model",
"common.oauth",
"health.cpu",
...
]
}

````

**Buraya ait olanlar:**

- Marka/ürün adları: `landing.brandName`, `common.social-github`
- Teknik terimler/kısaltmalar: "health.cpu", "mcpDashboard.pid", "settings.ai"
- ICU/format dizeleri: `apiManager.modelsCount`, `health.millisecondsShort`
- Yer tutucu değerleri: `providers.openaiBaseUrlPlaceholder`, `cliTools.baseUrlPlaceholder`
- Protokol adları: "common.http", "common.oauth", "providers.oauth2Label"
- Gezinme bölümleri: `sidebar.primarySection`, `sidebar.cliSection`

**Anahtar eklemek için:**`scripts/i18n/untranslatable-keys.json` dosyasındaki `keys' dizisini düzenleyin ve doğrulamayı yeniden çalıştırın.## CI Integration

## Best Practices

### When Editing Translations

1.**Her zaman önce `en.json`u düzenleyin**— bu gerçeğin kaynağıdır
2.**Yeni anahtarları tüm yerel ayarlara yaymak için `generate-multilang.mjs message`**komutunu çalıştırın
3.**Otomatik çevirileri inceleyin**— Google Çeviri nihai değil, bir başlangıç noktasıdır
4.**Taahhüt etmeden önce doğrulayın**- `python3 scripts/validate_translation.py quick -l <dil>`
5.**Anahtarın İngilizce kalması gerekiyorsa `untranslatable-keys.json`u güncelleyin**### Placeholder Safety

- ICU yer tutucuları (`{count}`, `{value}`, `{total}`, `{seconds}`) tam olarak korunmalıdır
- Çoğul biçimler (`{count, plural, one {# model} other {# models}}`) yapıyı korumalıdır
- Doğrulayıcı yer tutucu uyumsuzluklarını otomatik olarak algılar### Adding New Translation Keys in Code

```tsx
// Use namespaced keys
const t = useTranslations("settings");
t("cacheSettings"); // maps to settings.cacheSettings in JSON

// Run check_translations.py to verify keys exist
python3 scripts/check_translations.py --verbose
````

### RTL Considerations

- Arapça ('ar') ve İbranice ('he') RTL yerel ayarlarıdır
- Sabit kodlanmış 'sol'/'sağ' CSS'den kaçının; 'başlangıç'/'son' mantıksal özelliklerini kullanın
- Visual QA, RTL düzeni uyumsuzluklarını 'run-visual-qa.mjs' aracılığıyla yakalar## Known Issues & History

### `in.json` → `hi.json` Fix

Oluşturucu başlangıçta Hintçe için doğru ISO 639-1 "hi" yerine "code: "in" (kullanımdan kaldırılmış Google Çeviri kodu) kullanıyordu. Bu, 'hi.json'un artık bir 'in.json' kopyasını oluşturdu. `generate-multilang.mjs` dosyasındaki `code: "in"`in `code: "hi"` olarak değiştirilmesi ve artık dosyanın kaldırılmasıyla düzeltildi.### `docs/i18n/README.md` Is Auto-Generated

'docs/i18n/README.md' dosyası 'generate-multilang.mjs docs' tarafından tamamen yeniden oluşturuldu. Tüm manuel düzenlemeler kaybolacaktır. Kalıcı olması gereken elle yazılmış belgeler için 'docs/I18N.md'yi (bu dosya) kullanın.### External Untranslatable Keys List

"untranslatable-keys.json" izin verilenler listesi, daha kolay bakım için "validate_translation.py" içindeki satır içi Python kümesinden harici bir JSON dosyasına taşındı. Doğrulayıcı bunu çalışma zamanında yükler.### `generate-multilang.mjs` Hindi Code Fix

Oluşturucu başlangıçta Hintçe için doğru ISO 639-1 "hi" yerine "code: "in" (kullanımdan kaldırılmış Google Çeviri kodu) kullanıyordu. Bu, 'diegosouzapw' tarafından '952b0b22c' yukarı akış işleminde tanıtıldı. `LOCALE_SPECS` dizisinde `code: "in"`in `code: "hi"` olarak değiştirilmesi ve artık kullanılmayan `in.json` dosyasının kaldırılmasıyla düzeltildi.### `validate_translation.py` Ignored Count Output
