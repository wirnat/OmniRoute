### Source of Truth

-**UI 문자열**: `src/i18n/messages/en.json`(영어 소스, ~2800키) -**로캘 파일**: `src/i18n/messages/{locale}.json` (30개 번역) -**프레임워크**: 쿠키 기반 로케일 확인 기능을 갖춘 `next-intl` -**구성**: `src/i18n/config.ts` — 30개의 로케일, 언어 이름, 플래그를 모두 정의합니다.### Runtime Flow

1. 사용자가 언어를 선택 → 'NEXT_LOCALE' 쿠키 세트
2. `src/i18n/request.ts`는 로케일을 확인합니다: 쿠키 → `Accept-Language` 헤더 → 폴백 `en`
3. 동적 가져오기는 `messages/{locale}.json`을 로드합니다.
4. 구성요소는 `useTranslations("namespace")` 및 `t("key")`를 사용합니다.### Supported Locales

### 1. Register the Locale

`src/i18n/config.ts`를 편집하세요:```ts
// Add to LOCALES array
"xx",
// Add to LANGUAGES array
{ code: "xx", label: "XX", name: "Language Name", flag: "🏳️" },

````

### 2. Add to Generator

`scripts/i18n/generate-multilang.mjs`를 편집하고 `LOCALE_SPECS`에 항목을 추가합니다.```js
{
  code: "xx",
  googleTl: "xx",
  label: "XX",
  flag: "🏳️",
  languageName: "Language Name",
  readmeName: "Language Name",
  docsName: "Language Name",
},
````

### 3. Generate Initial Translation

```bash
node scripts/i18n/generate-multilang.mjs messages
```

그러면 Google 번역을 통해 `en.json`에서 자동 번역된 `src/i18n/messages/xx.json`이 생성됩니다.### 4. Review & Fix Auto-Translations

자동 번역은 출발점입니다. 다음 항목을 수동으로 검토하세요.

- 기술적 정확성
- 상황에 맞는 용어
- 자리 표시자(`{count}`, `{value}` 등)의 올바른 처리### 5. Validate

```bash
python3 scripts/validate_translation.py quick -l xx
python3 scripts/validate_translation.py diff common -l xx
```

### 6. Generate Translated Documentation

```bash
node scripts/i18n/generate-multilang.mjs docs
```

## Auto-Translation Pipeline

### generate-multilang.mjs (Google Translate)

**기본 자동 번역 엔진**— Google 번역 무료 API를 사용하여 UI 문자열, README 및 문서에 대한 번역을 생성합니다.```bash
node scripts/i18n/generate-multilang.mjs [messages|readme|docs|all]

````

| 모드 | 그것이 하는 일 |
| ---------- | --------------------------------------------------- |
| `메시지` | `en.json`에서 `src/i18n/messages/{locale}.json`에 누락된 키를 번역합니다 |
| `읽어보기` | 프로젝트 루트에서 `README.md`를 모든 로케일로 `README.{code}.md`로 변환 |
| `문서` | `DOC_SOURCE_FILES`를 `docs/i18n/{locale}/{docName}`으로 변환 |
| `모두` | 세 가지 모드를 모두 실행 |

**특징:**

-**텍스트 보호**: 번역 전에 코드 블록(` ``` `), 인라인 코드(`` ` `), 마크다운 링크/이미지(`[text](url)`), HTML 태그, 테이블 및 ICU 자리 표시자(`{count}`, `{value}`, `{total}` 등)를 마스킹한 후 복원합니다.
-**청크 일괄 처리**: `__OMNIROUTE_I18N_SEPARATOR__` 구분 기호를 사용하여 여러 문자열을 결합하여 API 호출을 최소화합니다(요청당 최대 1800자).
-**인메모리 캐시**: 세션 내에서 반복되는 문자열에 대한 중복 API 호출을 방지합니다.
-**재시도 로직**: 429/5xx 오류에 대한 지수 백오프(300ms × 시도 지연으로 최대 5회 시도)
-**시간 초과**: 요청당 20초
-**기존 건너뛰기**: 대상 파일이 이미 존재하는 경우 덮어쓰지 않습니다.

**중요한 행동:**

- `docs/i18n/README.md`는 실행될 때마다**재생성**됩니다. 이는 모든 문서의 자동 생성된 색인입니다.
- 루트 `README.{code}.md` 파일은 존재하지 않는 경우에만 생성됩니다(`EXISTING_README_CODES`의 로케일은 건너뜁니다).
- 번역된 모든 문서에 입력 도구 모음(`🌐**Languages:**...`)이 자동으로 삽입/업데이트됩니다.### i18n_autotranslate.py (LLM-based)

**보조 번역기**— OpenAI 호환 LLM API(OmniRoute 자체 포함)를 사용하여 기존 `docs/i18n/` 마크다운 파일을 번역합니다. Google 번역보다 더 나은 품질로 문서를 다듬거나 재번역하는 데 가장 적합합니다.```bash
python3 scripts/i18n_autotranslate.py \
  --api-url http://localhost:20128/v1 \
  --api-key sk-your-key \
  --model gpt-4o
````

**특징:**

- `docs/i18n/` 마크다운 파일에서 영어 단락을 검사합니다.
- 코드 블록, 테이블 및 이미 번역된 콘텐츠를 건너뜁니다.
- 기술 번역 시스템 프롬프트를 통해 LLM에 단락을 보냅니다.
- 30개 언어 모두 지원## Validation & QA

### validate_translation.py

**번역 검사기**— 모든 로케일 JSON을 'en.json'과 비교하고 문제를 보고합니다.```bash

# Quick check (counts only)

python3 scripts/validate_translation.py quick -l cs

# Output:

# Missing: 0

# Untranslated: 0

# Ignored (UNTRANSLATABLE_KEYS): 236

# Detailed diff by category

python3 scripts/validate_translation.py diff common -l cs
python3 scripts/validate_translation.py diff settings -l cs

# Export to CSV

python3 scripts/validate_translation.py csv -l cs > report.csv

# Export to Markdown

python3 scripts/validate_translation.py md -l cs > report.md

# Full report (default)

python3 scripts/validate_translation.py -l cs

````

**감지:**

-**누락된 키**— `en.json`에는 키가 있지만 로케일 파일에는 없습니다.
-**추가 키**— 로케일 파일에는 키가 있지만 `en.json`에는 없습니다.
-**번역되지 않은 키**— 로케일 값이 영어 소스와 동일한 키(허용 목록 제외)
-**자리 표시자 불일치**— 소스와 번역이 일치하지 않는 ICU 자리 표시자

**종료 코드:**
| 코드 | 의미 |
|------|---------|
| 0 | 알았어 |
| 1 | 일반 오류 |
| 2 | 문자열 누락(하드 오류) |
| 3 | 번역되지 않은 경고(소프트) |

**환경:**`TRANSLATION_LANG=cs`를 설정하거나 `-l cs` 플래그를 사용하세요.### check_translations.py

**Code-to-JSON 키 검사기**— `useTranslations()` 호출에 대해 `src/**/*.tsx` 및 `src/**/*.ts`를 검사하고 참조된 모든 키가 `en.json`에 있는지 확인합니다.```bash
# Basic check
python3 scripts/check_translations.py

# Verbose output
python3 scripts/check_translations.py --verbose

# Auto-fix (adds missing keys to en.json)
python3 scripts/check_translations.py --fix
````

### untranslatable-keys.json

**파일:**`scripts/i18n/untranslatable-keys.json`

영어 소스와 동일하게 유지되어야 하는 키의 허용 목록입니다. '번역되지 않음' 경고를 피하기 위해 `validate_translation.py`에서 사용됩니다.```json
{
"description": "Keys that should remain untranslated...",
"keys": [
"common.model",
"common.oauth",
"health.cpu",
...
]
}

````

**여기에 속하는 것:**

- 브랜드/제품 이름: `landing.brandName`, `common.social-github`
- 기술 용어/약어: `health.cpu`, `mcpDashboard.pid`, `settings.ai`
- ICU/형식 문자열: `apiManager.modelsCount`, `health.millisecondsShort`
- 자리 표시자 값: `providers.openaiBaseUrlPlaceholder`, `cliTools.baseUrlPlaceholder`
- 프로토콜 이름: `common.http`, `common.oauth`, `providers.oauth2Label`
- 탐색 섹션: `sidebar.primarySection`, `sidebar.cliSection`

**키를 추가하려면:**`scripts/i18n/untranslatable-keys.json`에서 `keys` 배열을 편집하고 유효성 검사를 다시 실행하세요.## CI Integration

## Best Practices

### When Editing Translations

1.**항상 `en.json`을 먼저 편집하세요**— 그것이 진실의 원천입니다
2.**`generate-multilang.mjs message`**를 실행하여 새 키를 모든 로케일에 전파합니다.
3.**자동 번역 검토**— Google 번역은 최종이 아닌 시작점입니다.
4.**커밋하기 전에 유효성 검사**— `python3 scripts/validate_translation.pyquick -l <lang>`
5.**키가 영어로 유지되어야 하는 경우 `untranslatable-keys.json`을 업데이트하세요.### Placeholder Safety

- ICU 자리 표시자(`{count}`, `{value}`, `{total}`, `{seconds}`)는 정확하게 보존되어야 합니다.
- 복수 형식(`{count, plural, one {# model} other {# models}}`)은 구조를 유지해야 합니다.
- 유효성 검사기는 자리 표시자 불일치를 자동으로 감지합니다.### Adding New Translation Keys in Code

```tsx
// Use namespaced keys
const t = useTranslations("settings");
t("cacheSettings"); // maps to settings.cacheSettings in JSON

// Run check_translations.py to verify keys exist
python3 scripts/check_translations.py --verbose
````

### RTL Considerations

- 아랍어(`ar`) 및 히브리어(`he`)는 RTL 로케일입니다.
