### Source of Truth

-**Șiruri UI**: `src/i18n/messages/en.json` (sursă în limba engleză, ~2800 de taste) -**Fișiere locale**: `src/i18n/messages/{locale}.json` (30 de traduceri) -**Cadru**: `next-intl` cu rezoluție locală bazată pe cookie-uri -**Config**: `src/i18n/config.ts` — definește toate cele 30 de localități, nume de limbă, steaguri### Runtime Flow

1. Utilizatorul selectează limba → set de cookie-uri `NEXT_LOCALE`
2. `src/i18n/request.ts` rezolvă locale: cookie → antetul `Accept-Language` → fallback `ro`
3. Importul dinamic încarcă `messages/{locale}.json`
4. Componentele folosesc `useTranslations("namespace")` și `t("key")`### Supported Locales

### 1. Register the Locale

Editați `src/i18n/config.ts`:```ts
// Add to LOCALES array
"xx",
// Add to LANGUAGES array
{ code: "xx", label: "XX", name: "Language Name", flag: "🏳️" },

````

### 2. Add to Generator

Editați `scripts/i18n/generate-multilang.mjs` — adăugați intrarea la `LOCALE_SPECS`:```js
{
  code: "xx",
  googleTl: "xx",
  label: "XX",
  flag: "🏳️",
  languageName: "Language Name",
  readmeName: "Language Name",
  docsName: "Language Name",
},
````

### 3. Generate Initial Translation

```bash
node scripts/i18n/generate-multilang.mjs messages
```

Acest lucru creează `src/i18n/messages/xx.json` tradus automat din `en.json` prin Google Translate.### 4. Review & Fix Auto-Translations

Traducerile automate sunt un punct de plecare. Examinați manual pentru:

- Precizie tehnică
- Terminologie adecvată contextului
- Gestionarea corectă a substituenților (`{count}`, `{value}` etc.)### 5. Validate

```bash
python3 scripts/validate_translation.py quick -l xx
python3 scripts/validate_translation.py diff common -l xx
```

### 6. Generate Translated Documentation

```bash
node scripts/i18n/generate-multilang.mjs docs
```

## Auto-Translation Pipeline

### generate-multilang.mjs (Google Translate)

**Motorul principal de traducere automată**— folosește API-ul gratuit Google Translate pentru a genera traduceri pentru șirurile de interfață de utilizare, README-uri și documentație.```bash
node scripts/i18n/generate-multilang.mjs [messages|readme|docs|all]

````

| Modul | Ce face |
| ---------- | ---------------------------------------------------------------------------- |
| `mesaje` | Traduce cheile lipsă în `src/i18n/messages/{locale}.json` din `en.json` |
| `citește-mă` | Traduce `README.md` în toate localurile ca `README.{code}.md` în rădăcina proiectului |
| `docs` | Traduce `DOC_SOURCE_FILES` în `docs/i18n/{locale}/{docName}` |
| `toate` | Rulează toate cele trei moduri |

**Caracteristici:**

-**Protecția textului**: maschează blocurile de cod (` ``` `), codul inline (`` ` ``), link-uri/imagini de reducere (`[text](url)`), etichete HTML, tabele și substituenți ICU (`{count}`, `{value}`, `{total}`, etc.) înainte de traducere, apoi le restaurează
-**Chunked loturi**: îmbină mai multe șiruri de caractere cu delimitatori `__OMNIROUTE_I18N_SEPARATOR__` pentru a minimiza apelurile API (maximum 1800 de caractere per solicitare)
-**Cache în memorie**: evită apelurile API redundante pentru șiruri repetate într-o sesiune
-**Reîncercați logica**: retragere exponențială (până la 5 încercări cu întârziere de 300 ms × încercare) pentru erori 429/5xx
-**Timeout**: 20 de secunde per solicitare
-**Omite existenta**: Dacă fișierul țintă există deja, acesta NU este suprascris

**Comportamente importante:**

- `docs/i18n/README.md` este**regenerat**la fiecare rulare — este un index generat automat pentru toate documentele
- Fișierele rădăcină `README.{code}.md` sunt create numai dacă nu există (omite localurile în `EXISTING_README_CODES`)
- Barele de limbă (`🌐**Limbi:**...`) sunt inserate/actualizate automat în toate documentele traduse### i18n_autotranslate.py (LLM-based)

**Translator secundar**— folosește orice API LLM compatibil cu OpenAI (inclusiv OmniRoute însuși) pentru a traduce fișierele existente `docs/i18n/` markdown. Cel mai bun pentru lustruirea sau retraducerea documentelor cu o calitate mai bună decât Google Translate.```bash
python3 scripts/i18n_autotranslate.py \
  --api-url http://localhost:20128/v1 \
  --api-key sk-your-key \
  --model gpt-4o
````

**Caracteristici:**

- Scanează fișierele markdown `docs/i18n/` pentru paragrafe în limba engleză
- Omite blocurile de cod, tabelele și conținutul deja tradus
- Trimite paragrafe la LLM cu promptul sistemului de traducere tehnică
- Acceptă toate cele 30 de limbi## Validation & QA

### validate_translation.py

**Validator de traducere**— compară orice local JSON cu `en.json` și raportează probleme.```bash

# Quick check (counts only)

python3 scripts/validate_translation.py quick -l cs

# Output:

# Missing: 0

# Untranslated: 0

# Ignored (UNTRANSLATABLE_KEYS): 236

# Detailed diff by category

python3 scripts/validate_translation.py diff common -l cs
python3 scripts/validate_translation.py diff settings -l cs

# Export to CSV

python3 scripts/validate_translation.py csv -l cs > report.csv

# Export to Markdown

python3 scripts/validate_translation.py md -l cs > report.md

# Full report (default)

python3 scripts/validate_translation.py -l cs

````

**Detectă:**

-**Cheile lipsă**— chei în `en.json`, dar nu în fișierul local
-**Chei suplimentare**— chei în fișierul local, dar nu în `en.json`
-**Chei netraduse**— chei în care valoarea localității este egală cu sursa engleză (excluzând lista permisă)
-**Nepotriviri de substituenți**— substituenți de UCI care nu se potrivesc între sursă și traducere

**Coduri de ieșire:**
| Cod | Înțeles |
|------|---------|
| 0 | OK |
| 1 | Eroare generică |
| 2 | Lipsesc șiruri de caractere (eroare grea) |
| 3 | Avertisment netradus (soft) |

**Mediu:**Setați `TRANSLATION_LANG=cs` sau utilizați indicatorul `-l cs`.### check_translations.py

**Verificator de chei Code-to-JSON**— scanează `src/**/*.tsx` și `src/**/*.ts` pentru apeluri `useTranslations()` și verifică că toate cheile la care se face referire există în `en.json`.```bash
# Basic check
python3 scripts/check_translations.py

# Verbose output
python3 scripts/check_translations.py --verbose

# Auto-fix (adds missing keys to en.json)
python3 scripts/check_translations.py --fix
````

**Analiză statică QA**— scanează fișierele de pagină Next.js pentru valorile de risc i18n și generează un raport Markdown.```bash

**Controale:**

**Ieșire:**`docs/reports/i18n-qa-checklist-{date}.md`### run-visual-qa.mjs

**Ieșire:**`docs/reports/i18n-visual-qa-{date}.md` + raport JSON## Managing Untranslatable Keys

### untranslatable-keys.json

**Fișier:**`scripts/i18n/untranslatable-keys.json`

Lista permisă de chei care ar trebui să rămână identice cu sursa engleză. Folosit de `validate_translation.py` pentru a evita avertismentele fals pozitive „netraduse”.```json
{
"description": "Keys that should remain untranslated...",
"keys": [
"common.model",
"common.oauth",
"health.cpu",
...
]
}

````

**Ceea ce este aici:**

- Nume de marcă/produs: `landing.brandName`, `common.social-github`
- Termeni/acronime tehnici: `health.cpu`, `mcpDashboard.pid`, `settings.ai`
- ICU/șiruri de format: `apiManager.modelsCount`, `health.millisecondsShort`
- Valori substituenți: `providers.openaiBaseUrlPlaceholder`, `cliTools.baseUrlPlaceholder`
- Nume de protocol: `common.http`, `common.oauth`, `providers.oauth2Label`
- Secțiuni de navigare: `sidebar.primarySection`, `sidebar.cliSection`

**Pentru a adăuga o cheie:**Editați matricea `keys` în `scripts/i18n/untranslatable-keys.json` și executați din nou validarea.## CI Integration

## Best Practices

### When Editing Translations

1.**Editați întotdeauna `en.json` mai întâi**— este sursa adevărului
2.**Rulați `generate-multilang.mjs messages`**pentru a propaga chei noi în toate localurile
3.**Examinați traducerile automate**— Google Translate este un punct de plecare, nu final
4.**Validați înainte de comitere**— `python3 scripts/validate_translation.py quick -l <lang>`
5.**Actualizați `untranslatable-keys.json`**dacă o cheie ar trebui să rămână în engleză### Placeholder Safety

- Substituenții ICU (`{count}`, `{value}`, `{total}`, `{secunde}`) trebuie păstrați exact
- Formatele plurale (`{count, plural, one {# model} other {# models}}`) trebuie să mențină structura
- Validatorul detectează automat nepotrivirile de substituent### Adding New Translation Keys in Code

```tsx
// Use namespaced keys
const t = useTranslations("settings");
t("cacheSettings"); // maps to settings.cacheSettings in JSON

// Run check_translations.py to verify keys exist
python3 scripts/check_translations.py --verbose
````

### RTL Considerations

- Arabă (`ar`) și ebraică (`he`) sunt localități RTL
- Evitați CSS-ul `left`/`right` codificat - folosiți proprietățile logice `start`/`end`
- Visual QA detectează nepotrivirile aspectului RTL prin `run-visual-qa.mjs`## Known Issues & History

Generatorul a folosit inițial „cod: „în”” (cod Google Translate demodat) pentru hindi în loc de ISO 639-1 corect „hi”. Aceasta a creat o copie orfană „in.json” a lui „hi.json”.
