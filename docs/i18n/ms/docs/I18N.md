### Source of Truth

-**Rentetan UI**: `src/i18n/messages/en.json` (sumber bahasa Inggeris, ~2800 kekunci) -**Fail tempatan**: `src/i18n/messages/{locale}.json` (30 terjemahan) -**Rangka Kerja**: `next-intl` dengan peleraian setempat berasaskan kuki -**Config**: `src/i18n/config.ts` — mentakrifkan kesemua 30 tempat, nama bahasa, bendera### Runtime Flow

1. Pengguna memilih bahasa → set kuki `NEXT_LOCALE`
2. `src/i18n/request.ts` menyelesaikan tempat: kuki → pengepala `Terima-Bahasa` → sandaran `en`
3. Import dinamik memuatkan `message/{locale}.json`
4. Komponen menggunakan `useTranslations("namespace")` dan `t("key")`### Supported Locales

### 1. Register the Locale

Edit `src/i18n/config.ts`:```ts
// Add to LOCALES array
"xx",
// Add to LANGUAGES array
{ code: "xx", label: "XX", name: "Language Name", flag: "🏳️" },

````

### 2. Add to Generator

Edit `scripts/i18n/generate-multilang.mjs` — tambah entri pada `LOCALE_SPECS`:```js
{
  code: "xx",
  googleTl: "xx",
  label: "XX",
  flag: "🏳️",
  languageName: "Language Name",
  readmeName: "Language Name",
  docsName: "Language Name",
},
````

### 3. Generate Initial Translation

```bash
node scripts/i18n/generate-multilang.mjs messages
```

Ini mencipta `src/i18n/messages/xx.json` auto-terjemah daripada `en.json` melalui Terjemahan Google.### 4. Review & Fix Auto-Translations

Autoterjemah ialah titik permulaan. Semak secara manual untuk:

- Ketepatan teknikal
- Terminologi yang sesuai dengan konteks
- Pengendalian ruang letak yang betul (`{count}`, `{value}`, dsb.)### 5. Validate

```bash
python3 scripts/validate_translation.py quick -l xx
python3 scripts/validate_translation.py diff common -l xx
```

### 6. Generate Translated Documentation

```bash
node scripts/i18n/generate-multilang.mjs docs
```

## Auto-Translation Pipeline

### generate-multilang.mjs (Google Translate)

**Enjin autoterjemah utama**— menggunakan API percuma Terjemahan Google untuk menjana terjemahan untuk rentetan UI, README dan dokumentasi.```bash
node scripts/i18n/generate-multilang.mjs [messages|readme|docs|all]

````

| Mod | Apa yang dilakukannya |
| ---------- | ----------------------------------------------------------------------------------- |
| `mesej` | Menterjemah kekunci yang hilang dalam `src/i18n/messages/{locale}.json` daripada `en.json` |
| `baca saya` | Terjemahkan `README.md` ke dalam semua tempat sebagai `README.{code}.md` dalam akar projek |
| `dokumen` | Menterjemah `DOC_SOURCE_FILES` ke dalam `docs/i18n/{locale}/{docName}` |
| `semua` | Menjalankan ketiga-tiga mod |

**Ciri-ciri:**

-**Perlindungan teks**: Sekatan kod topeng (` ``` `), kod sebaris (`` ` ``), pautan/imej markdown (`[teks](url)`), teg HTML, jadual dan ruang letak ICU (`{count}`, `{value}`, `{total}`, dsb.) sebelum terjemahan, kemudian memulihkannya
-**Chunked batching**: Menyertai berbilang rentetan dengan pembatas `__OMNIROUTE_I18N_SEPARATOR__` untuk meminimumkan panggilan API (maks 1800 aksara setiap permintaan)
-**Cache dalam memori**: Mengelakkan panggilan API berlebihan untuk rentetan berulang dalam sesi
-**Cuba semula logik**: Penyingkiran eksponen (sehingga 5 percubaan dengan kelewatan 300ms × percubaan) untuk ralat 429/5xx
-**Tamat masa**: 20 saat setiap permintaan
-**Langkau sedia ada**: Jika fail sasaran sudah wujud, ia TIDAK ditimpa

**Tingkah laku penting:**

- `docs/i18n/README.md`**dijana semula**setiap larian — ia ialah indeks yang dijana secara automatik bagi semua dokumen
- Fail `README.{code}.md` Root hanya dibuat jika ia tidak wujud (langkau tempat dalam `EXISTING_README_CODES`)
- Bar bahasa (`🌐**Bahasa:**...`) dimasukkan/dikemas kini secara automatik dalam semua dokumen terjemahan### i18n_autotranslate.py (LLM-based)

**Penterjemah kedua**— menggunakan mana-mana API LLM yang serasi dengan OpenAI (termasuk OmniRoute sendiri) untuk menterjemah fail penurunan nilai `docs/i18n/` sedia ada. Terbaik untuk menggilap atau menterjemah semula dokumen dengan kualiti yang lebih baik daripada Terjemahan Google.```bash
python3 scripts/i18n_autotranslate.py \
  --api-url http://localhost:20128/v1 \
  --api-key sk-your-key \
  --model gpt-4o
````

**Ciri-ciri:**

- Mengimbas fail markdown `docs/i18n/` untuk perenggan bahasa Inggeris
- Melangkau blok kod, jadual dan kandungan yang sudah diterjemahkan
- Menghantar perenggan ke LLM dengan sistem terjemahan teknikal segera
- Menyokong semua 30 bahasa## Validation & QA

### validate_translation.py

**Pengesah terjemahan**— membandingkan mana-mana JSON tempat dengan `en.json` dan melaporkan isu.```bash

# Quick check (counts only)

python3 scripts/validate_translation.py quick -l cs

# Output:

# Missing: 0

# Untranslated: 0

# Ignored (UNTRANSLATABLE_KEYS): 236

# Detailed diff by category

python3 scripts/validate_translation.py diff common -l cs
python3 scripts/validate_translation.py diff settings -l cs

# Export to CSV

python3 scripts/validate_translation.py csv -l cs > report.csv

# Export to Markdown

python3 scripts/validate_translation.py md -l cs > report.md

# Full report (default)

python3 scripts/validate_translation.py -l cs

````

**Mengesan:**

-**Kunci hilang**— kunci dalam `en.json` tetapi bukan dalam fail setempat
-**Kunci tambahan**— kunci dalam fail setempat tetapi bukan dalam `en.json`
-**Kunci tidak diterjemahkan**— kunci dengan nilai tempat bersamaan dengan sumber bahasa Inggeris (tidak termasuk senarai dibenarkan)
-**Ketakpadanan pemegang tempat**— Pemegang tempat ICU yang tidak sepadan antara sumber dan terjemahan

**Kod keluar:**
| Kod | Maksudnya |
|------|---------|
| 0 | OK |
| 1 | Ralat generik |
| 2 | Rentetan tiada (ralat sukar) |
| 3 | Amaran tidak diterjemahkan (lembut) |

**Persekitaran:**Tetapkan `TRANSLATION_LANG=cs` atau gunakan bendera `-l cs`.### check_translations.py

**Pemeriksa kunci kod-ke-JSON**— mengimbas `src/**/*.tsx` dan `src/**/*.ts` untuk panggilan `useTranslations()` dan mengesahkan semua kekunci yang dirujuk wujud dalam `en.json`.```bash
# Basic check
python3 scripts/check_translations.py

# Verbose output
python3 scripts/check_translations.py --verbose

# Auto-fix (adds missing keys to en.json)
python3 scripts/check_translations.py --fix
````

### untranslatable-keys.json

**Fail:**`scripts/i18n/untranslatable-keys.json`

Benarkan senarai kunci yang harus kekal sama dengan sumber bahasa Inggeris. Digunakan oleh `validate_translation.py` untuk mengelakkan amaran "tidak diterjemahkan" positif palsu.```json
{
"description": "Keys that should remain untranslated...",
"keys": [
"common.model",
"common.oauth",
"health.cpu",
...
]
}

````

**Apa yang ada di sini:**

- Nama jenama/produk: `landing.brandName`, `common.social-github`
- Istilah teknikal/akronim: `health.cpu`, `mcpDashboard.pid`, `settings.ai`
- Rentetan ICU/format: `apiManager.modelsCount`, `health.millisecondsShort`
- Nilai pemegang tempat: `providers.openaiBaseUrlPlaceholder`, `cliTools.baseUrlPlaceholder`
- Nama protokol: `common.http`, `common.oauth`, `providers.oauth2Label`
- Bahagian navigasi: `sidebar.primarySection`, `sidebar.cliSection`

**Untuk menambah kunci:**Edit tatasusunan `keys` dalam `scripts/i18n/untranslatable-keys.json` dan jalankan semula pengesahan.## CI Integration

## Best Practices

### When Editing Translations

1.**Sentiasa edit `en.json` dahulu**— ia adalah sumber kebenaran
2.**Jalankan `generate-multilang.mjs messages`**untuk menyebarkan kunci baharu ke semua tempat
3.**Semak terjemahan automatik**— Terjemahan Google ialah titik permulaan, bukan muktamad
4.**Sahkan sebelum melakukan**— `python3 scripts/validate_translation.py quick -l <lang>`
5.**Kemas kini `untranslatable-keys.json`**jika kunci harus kekal dalam bahasa Inggeris### Placeholder Safety

- Pemegang tempat ICU (`{count}`, `{value}`, `{total}`, `{seconds}`) mesti dikekalkan dengan tepat
- Format jamak (`{count, plural, one {# model} other {# models}}`) mesti mengekalkan struktur
- Pengesah mengesan ketidakpadanan pemegang tempat secara automatik### Adding New Translation Keys in Code

```tsx
// Use namespaced keys
const t = useTranslations("settings");
t("cacheSettings"); // maps to settings.cacheSettings in JSON

// Run check_translations.py to verify keys exist
python3 scripts/check_translations.py --verbose
````

### RTL Considerations

- Bahasa Arab (`ar`) dan Ibrani (`dia`) ialah tempat RTL
- Elakkan kod keras `kiri`/`kanan` CSS — gunakan sifat logik `mula`/`akhir`
- QA Visual menangkap ketidakpadanan reka letak RTL melalui `run-visual-qa.mjs`## Known Issues & History

### `in.json` → `hi.json` Fix

Penjana pada asalnya menggunakan `kod: "in"` (kod Terjemahan Google yang ditamatkan) untuk Hindi dan bukannya ISO 639-1 `hi` yang betul. Ini mencipta pendua `in.json` yatim piatu daripada `hi.json`.
