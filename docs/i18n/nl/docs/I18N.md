### Source of Truth

-**UI-strings**: `src/i18n/messages/en.json` (Engelse bron, ~2800 sleutels) -**Lokale bestanden**: `src/i18n/messages/{locale}.json` (30 vertalingen) -**Framework**: `next-intl` met op cookies gebaseerde landinstellingen -**Config**: `src/i18n/config.ts` — definieert alle 30 landinstellingen, taalnamen, vlaggen### Runtime Flow

1. Gebruiker selecteert taal → `NEXT_LOCALE` cookie-set
2. `src/i18n/request.ts` lost de landinstelling op: cookie → `Accept-Language` header → fallback `en`
3. Dynamische import laadt `messages/{locale}.json`
4. Componenten gebruiken `useTranslations("namespace")` en `t("key")`### Supported Locales

### 1. Register the Locale

Bewerk `src/i18n/config.ts`:```ts
// Add to LOCALES array
"xx",
// Add to LANGUAGES array
{ code: "xx", label: "XX", name: "Language Name", flag: "🏳️" },

````

### 2. Add to Generator

Bewerk `scripts/i18n/generate-multilang.mjs` — voeg item toe aan `LOCALE_SPECS`:```js
{
  code: "xx",
  googleTl: "xx",
  label: "XX",
  flag: "🏳️",
  languageName: "Language Name",
  readmeName: "Language Name",
  docsName: "Language Name",
},
````

### 3. Generate Initial Translation

```bash
node scripts/i18n/generate-multilang.mjs messages
```

Hierdoor wordt `src/i18n/messages/xx.json` automatisch vertaald vanuit `en.json` via Google Translate.### 4. Review & Fix Auto-Translations

Automatische vertalingen zijn een startpunt. Handmatig beoordelen op:

- Technische nauwkeurigheid
- Contextafhankelijke terminologie
- Correcte afhandeling van tijdelijke aanduidingen (`{count}`, `{value}`, etc.)### 5. Validate

```bash
python3 scripts/validate_translation.py quick -l xx
python3 scripts/validate_translation.py diff common -l xx
```

### 6. Generate Translated Documentation

```bash
node scripts/i18n/generate-multilang.mjs docs
```

## Auto-Translation Pipeline

### generate-multilang.mjs (Google Translate)

**Primaire automatische vertaalengine**: gebruikt de gratis API van Google Translate om vertalingen te genereren voor UI-tekenreeksen, README's en documentatie.```bash
node scripts/i18n/generate-multilang.mjs [messages|readme|docs|all]

````

| Modus | Wat het doet |
| ---------- | --------------------------------------------------------------------- |
| `berichten` | Vertaalt ontbrekende sleutels in `src/i18n/messages/{locale}.json` van `en.json` |
| `lees mij` | Vertaalt `README.md` naar alle landinstellingen als `README.{code}.md` in de hoofdmap van het project |
| `documenten` | Vertaalt `DOC_SOURCE_FILES` naar `docs/i18n/{locale}/{docName}` |
| `alle` | Voert alle drie de modi uit |

**Kenmerken:**

-**Tekstbeveiliging**: maskeert codeblokken (` ``` `), inline code (`` ` ``), markdown-links/afbeeldingen (`[text](url)`), HTML-tags, tabellen en ICU-placeholders (`{count}`, `{value}`, `{total}`, etc.) vóór vertaling, en herstelt deze vervolgens
-**Gedeelte batching**: voegt meerdere tekenreeksen samen met `__OMNIROUTE_I18N_SEPARATOR__` scheidingstekens om API-aanroepen te minimaliseren (max. 1800 tekens per verzoek)
-**In-memory cache**: vermijdt redundante API-aanroepen voor herhaalde tekenreeksen binnen een sessie
-**Logica voor opnieuw proberen**: Exponentiële uitstel (maximaal 5 pogingen met een vertraging van 300 ms × poging) voor 429/5xx-fouten
-**Time-out**: 20 seconden per verzoek
-**Bestaand overslaan**: als het doelbestand al bestaat, wordt het NIET overschreven

**Belangrijk gedrag:**

- `docs/i18n/README.md` wordt bij elke run**geregenereerd**— het is een automatisch gegenereerde index van alle documenten
- Root `README.{code}.md`-bestanden worden alleen aangemaakt als ze niet bestaan (slaat locales in `EXISTING_README_CODES` over)
- Taalbalken (`🌐**Talen:**...`) worden automatisch ingevoegd/bijgewerkt in alle vertaalde documenten### i18n_autotranslate.py (LLM-based)

**Secundaire vertaler**— gebruikt elke OpenAI-compatibele LLM API (inclusief OmniRoute zelf) om bestaande `docs/i18n/` markdown-bestanden te vertalen. Het beste voor het polijsten of opnieuw vertalen van documenten met een betere kwaliteit dan Google Translate.```bash
python3 scripts/i18n_autotranslate.py \
  --api-url http://localhost:20128/v1 \
  --api-key sk-your-key \
  --model gpt-4o
````

**Kenmerken:**

- Scant `docs/i18n/` markdown-bestanden op Engelse paragrafen
- Slaat codeblokken, tabellen en reeds vertaalde inhoud over
- Stuurt paragrafen naar LLM met een prompt voor een technisch vertaalsysteem
- Ondersteunt alle 30 talen## Validation & QA

### validate_translation.py

**Vertalingsvalidator**— vergelijkt elke lokale JSON met `en.json` en rapporteert problemen.```bash

# Quick check (counts only)

python3 scripts/validate_translation.py quick -l cs

# Output:

# Missing: 0

# Untranslated: 0

# Ignored (UNTRANSLATABLE_KEYS): 236

# Detailed diff by category

python3 scripts/validate_translation.py diff common -l cs
python3 scripts/validate_translation.py diff settings -l cs

# Export to CSV

python3 scripts/validate_translation.py csv -l cs > report.csv

# Export to Markdown

python3 scripts/validate_translation.py md -l cs > report.md

# Full report (default)

python3 scripts/validate_translation.py -l cs

````

**Detecteert:**

-**Ontbrekende sleutels**— sleutels in `en.json` maar niet in het localebestand
-**Extra sleutels**— sleutels in locale-bestand maar niet in `en.json`
-**Niet-vertaalde sleutels**: sleutels waarbij de landinstelling gelijk is aan de Engelse bron (exclusief toelatingslijst)
-**Placeholders komen niet overeen**— ICU-placeholders die niet overeenkomen tussen bron en vertaling

**Exitcodes:**
| Code | Betekenis |
|------|---------|
| 0 | Oké |
| 1 | Algemene fout |
| 2 | Ontbrekende tekenreeksen (harde fout) |
| 3 | Onvertaalde waarschuwing (zacht) |

**Omgeving:**Stel `TRANSLATION_LANG=cs` in of gebruik de vlag `-l cs`.### check_translations.py

**Code-naar-JSON-sleutelcontrole**— scant `src/**/*.tsx` en `src/**/*.ts` op `useTranslations()`-aanroepen en verifieert dat alle sleutels waarnaar wordt verwezen bestaan ​​in `en.json`.```bash
# Basic check
python3 scripts/check_translations.py

# Verbose output
python3 scripts/check_translations.py --verbose

# Auto-fix (adds missing keys to en.json)
python3 scripts/check_translations.py --fix
````

### generate-qa-checklist.mjs

**Statische analyse QA**— scant Next.js-paginabestanden op i18n-risicostatistieken en genereert een Markdown-rapport.```bash

**Cheques:**

- Klassengebruik met vaste breedte (overlooprisico)

**Uitvoer:**`docs/reports/i18n-qa-checklist-{date}.md`### run-visual-qa.mjs

**Uitvoer:**`docs/reports/i18n-visual-qa-{date}.md` + JSON-rapport## Managing Untranslatable Keys

### untranslatable-keys.json

**Bestand:**`scripts/i18n/untranslatable-keys.json`

Toegestane lijst met sleutels die identiek moeten blijven aan de Engelse bron. Gebruikt door `validate_translation.py` om vals-positieve "niet-vertaalde" waarschuwingen te voorkomen.```json
{
"description": "Keys that should remain untranslated...",
"keys": [
"common.model",
"common.oauth",
"health.cpu",
...
]
}

````

**Wat hoort hier:**

- Merk-/productnamen: `landing.brandName`, `common.social-github`
- Technische termen/acroniemen: `health.cpu`, `mcpDashboard.pid`, `settings.ai`
- ICU/format-tekenreeksen: `apiManager.modelsCount`, `health.millisecondsShort`
- Placeholder-waarden: `providers.openaiBaseUrlPlaceholder`, `cliTools.baseUrlPlaceholder`
- Protocolnamen: `common.http`, `common.oauth`, `providers.oauth2Label`
- Navigatiesecties: `sidebar.primarySection`, `sidebar.cliSection`

**Om een sleutel toe te voegen:**Bewerk de `keys` array in `scripts/i18n/untranslatable-keys.json` en voer de validatie opnieuw uit.## CI Integration

## Best Practices

### When Editing Translations

1.**Bewerk `en.json` altijd eerst**— dit is de bron van de waarheid
2.**Voer `generate-multilang.mjs messages`**uit om nieuwe sleutels naar alle landinstellingen door te geven
3.**Automatische vertalingen beoordelen**— Google Translate is een startpunt, niet het eindpunt
4.**Valideren voordat je commit**— `python3 scripts/validate_translation.py quick -l <lang>`
5.**Update `untranslatable-keys.json`**als een sleutel in het Engels moet blijven### Placeholder Safety

- ICU-placeholders (`{count}`, `{value}`, `{total}`, `{seconds}`) moeten exact behouden blijven
- Meervoudsformaten (`{count, plural, one {# model} other {# models}}`) moeten de structuur behouden
- De validator detecteert automatisch niet-overeenkomende placeholders### Adding New Translation Keys in Code

```tsx
// Use namespaced keys
const t = useTranslations("settings");
t("cacheSettings"); // maps to settings.cacheSettings in JSON

// Run check_translations.py to verify keys exist
python3 scripts/check_translations.py --verbose
````

### RTL Considerations

- Arabisch (`ar`) en Hebreeuws (`he`) zijn RTL-landinstellingen
- Vermijd hardgecodeerde `left`/`right` CSS - gebruik `start`/`end` logische eigenschappen
- Visuele QA detecteert niet-overeenkomende RTL-indelingen via `run-visual-qa.mjs`## Known Issues & History

### `in.json` → `hi.json` Fix

De generator gebruikte oorspronkelijk `code: "in"` (verouderde Google Translate-code) voor Hindi in plaats van de juiste ISO 639-1 `hi`. Hierdoor werd een verweesd `in.json`-duplicaat van `hi.json` gemaakt. Opgelost door `code: "in"` te veranderen in `code: "hi"` in `generate-multilang.mjs` en het zwevende bestand te verwijderen.### `docs/i18n/README.md` Is Auto-Generated

Het bestand `docs/i18n/README.md` wordt volledig opnieuw gegenereerd door `generate-multilang.mjs docs`. Eventuele handmatige bewerkingen gaan verloren. Gebruik `docs/I18N.md` (dit bestand) voor handgeschreven documentatie die moet blijven bestaan.### External Untranslatable Keys List

De toelatingslijst `untranslatable-keys.json` is verplaatst van een inline Python-set in `validate_translation.py` naar een extern JSON-bestand voor eenvoudiger onderhoud. De validator laadt het tijdens runtime.### `generate-multilang.mjs` Hindi Code Fix

De generator gebruikte oorspronkelijk `code: "in"` (verouderde Google Translate-code) voor Hindi in plaats van de juiste ISO 639-1 `hi`. Dit werd geïntroduceerd in upstream commit `952b0b22c` door `diegosouzapw`. Opgelost door `code: "in"` te veranderen in `code: "hi"` in de `LOCALE_SPECS` array en het verweesde `in.json`-bestand te verwijderen.### `validate_translation.py` Ignored Count Output
