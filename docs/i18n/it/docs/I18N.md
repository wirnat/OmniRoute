### Source of Truth

-**Stringhe dell'interfaccia utente**: `src/i18n/messages/en.json` (fonte inglese, ~2800 chiavi) -**File locali**: `src/i18n/messages/{locale}.json` (30 traduzioni) -**Framework**: `next-intl` con risoluzione locale basata su cookie -**Config**: `src/i18n/config.ts` — definisce tutte le 30 localizzazioni, nomi delle lingue e flag### Runtime Flow

1. L'utente seleziona la lingua → set di cookie `NEXT_LOCALE`
2. `src/i18n/request.ts` risolve la locale: cookie → intestazione `Accept-Language` → fallback `en`
3. L'importazione dinamica carica `messages/{locale}.json`
4. I componenti utilizzano `useTranslations("namespace")` e `t("key")`### Supported Locales

### 1. Register the Locale

Modifica `src/i18n/config.ts`:```ts
// Add to LOCALES array
"xx",
// Add to LANGUAGES array
{ code: "xx", label: "XX", name: "Language Name", flag: "🏳️" },

````

### 2. Add to Generator

Modifica `scripts/i18n/generate-multilang.mjs` — aggiungi la voce a `LOCALE_SPECS`:```js
{
  code: "xx",
  googleTl: "xx",
  label: "XX",
  flag: "🏳️",
  languageName: "Language Name",
  readmeName: "Language Name",
  docsName: "Language Name",
},
````

### 3. Generate Initial Translation

```bash
node scripts/i18n/generate-multilang.mjs messages
```

Questo crea `src/i18n/messages/xx.json` tradotto automaticamente da `en.json` tramite Google Translate.### 4. Review & Fix Auto-Translations

Le traduzioni automatiche sono un punto di partenza. Rivedi manualmente per:

- Precisione tecnica
- Terminologia appropriata al contesto
- Gestione corretta dei segnaposto (`{count}`, `{value}`, ecc.)### 5. Validate

```bash
python3 scripts/validate_translation.py quick -l xx
python3 scripts/validate_translation.py diff common -l xx
```

### 6. Generate Translated Documentation

```bash
node scripts/i18n/generate-multilang.mjs docs
```

## Auto-Translation Pipeline

### generate-multilang.mjs (Google Translate)

**Motore di traduzione automatica primario**: utilizza l'API gratuita di Google Translate per generare traduzioni per stringhe dell'interfaccia utente, README e documentazione.```bash
node scripts/i18n/generate-multilang.mjs [messages|readme|docs|all]

````

| Modalità | Cosa fa |
| ---------- | ----------------------------------------------------------------------------- |
| `messaggi` | Traduce le chiavi mancanti in `src/i18n/messages/{locale}.json` da `en.json` |
| `leggimi` | Traduce `README.md` in tutte le versioni locali come `README.{code}.md` nella root del progetto |
| `documenti` | Traduce `DOC_SOURCE_FILES` in `docs/i18n/{locale}/{docName}` |
| "tutti" | Esegue tutte e tre le modalità |

**Caratteristiche:**

-**Protezione del testo**: maschera i blocchi di codice (` ``` `), il codice in linea (`` ` ``), link/immagini di markdown (`[text](url)`), tag HTML, tabelle e segnaposto ICU (`{count}`, `{value}`, `{total}`, ecc.) prima della traduzione, quindi li ripristina
-**Batch in blocchi**: unisce più stringhe con delimitatori `__OMNIROUTE_I18N_SEPARATOR__` per ridurre al minimo le chiamate API (max 1800 caratteri per richiesta)
-**Cache in memoria**: evita chiamate API ridondanti per stringhe ripetute all'interno di una sessione
-**Logica dei tentativi**: backoff esponenziale (fino a 5 tentativi con 300 ms × ritardo tentativo) per errori 429/5xx
-**Timeout**: 20 secondi per richiesta
-**Salta esistente**: se il file di destinazione esiste già, NON viene sovrascritto

**Comportamenti importanti:**

- `docs/i18n/README.md` viene**rigenerato**a ogni esecuzione: è un indice generato automaticamente di tutti i documenti
- I file root `README.{code}.md` vengono creati solo se non esistono (salta le localizzazioni in `EXISTING_README_CODES`)
- Le barre della lingua (`🌐**Lingue:**...`) vengono inserite/aggiornate automaticamente in tutti i documenti tradotti### i18n_autotranslate.py (LLM-based)

**Traduttore secondario**: utilizza qualsiasi API LLM compatibile con OpenAI (incluso lo stesso OmniRoute) per tradurre i file markdown `docs/i18n/` esistenti. Ideale per perfezionare o ritradurre documenti con una qualità migliore rispetto a Google Translate.```bash
python3 scripts/i18n_autotranslate.py \
  --api-url http://localhost:20128/v1 \
  --api-key sk-your-key \
  --model gpt-4o
````

**Caratteristiche:**

- Esegue la scansione dei file markdown `docs/i18n/` per i paragrafi in inglese
- Salta blocchi di codice, tabelle e contenuti già tradotti
- Invia paragrafi a LLM con il prompt del sistema di traduzione tecnica
- Supporta tutte le 30 lingue## Validation & QA

### validate_translation.py

**Convalidatore di traduzione**: confronta qualsiasi JSON locale con "en.json" e segnala i problemi.```bash

# Quick check (counts only)

python3 scripts/validate_translation.py quick -l cs

# Output:

# Missing: 0

# Untranslated: 0

# Ignored (UNTRANSLATABLE_KEYS): 236

# Detailed diff by category

python3 scripts/validate_translation.py diff common -l cs
python3 scripts/validate_translation.py diff settings -l cs

# Export to CSV

python3 scripts/validate_translation.py csv -l cs > report.csv

# Export to Markdown

python3 scripts/validate_translation.py md -l cs > report.md

# Full report (default)

python3 scripts/validate_translation.py -l cs

````

**Rileva:**

-**Chiavi mancanti**: chiavi in `en.json` ma non nel file locale
-**Chiavi extra**: chiavi nel file locale ma non in `en.json`
-**Chiavi non tradotte**: chiavi in cui il valore locale è uguale all'origine inglese (esclusa la lista consentita)
-**Mancate corrispondenze dei segnaposto**: segnaposto ICU che non corrispondono tra fonte e traduzione

**Codici di uscita:**
| Codice | Significato |
|------|---------|
| 0| Va bene |
| 1| Errore generico |
| 2| Stringhe mancanti (errore grave) |
| 3| Avviso non tradotto (soft) |

**Ambiente:**imposta `TRANSLATION_LANG=cs` o utilizza il flag `-l cs`.### check_translations.py

**Controllo chiavi Code-to-JSON**: analizza `src/**/*.tsx` e `src/**/*.ts` per le chiamate `useTranslations()` e verifica che tutte le chiavi di riferimento esistano in `en.json`.```bash
# Basic check
python3 scripts/check_translations.py

# Verbose output
python3 scripts/check_translations.py --verbose

# Auto-fix (adds missing keys to en.json)
python3 scripts/check_translations.py --fix
````

### generate-qa-checklist.mjs

**QA di analisi statica**: analizza i file di pagina Next.js per le metriche di rischio i18n e genera un report Markdown.```bash
node scripts/i18n/generate-qa-checklist.mjs

````

**Assegni:**

- Utilizzo delle classi a larghezza fissa (rischio di overflow)
- Classi direzionali sinistra/destra (rischio RTL)
- Modelli soggetti a ritaglio
- Parità locale (chiavi mancanti/extra rispetto a `en.json`)
- Barre di selezione della lingua README nelle impostazioni locali prioritarie (`es`, `fr`, `de`, `ja`, `ar`)

**Risultato:**`docs/reports/i18n-qa-checklist-{date}.md`### run-visual-qa.mjs

**Output:**`docs/reports/i18n-visual-qa-{date}.md` + rapporto JSON## Managing Untranslatable Keys

### untranslatable-keys.json

**File:**`scripts/i18n/untranslatable-keys.json`

Elenco consentito di chiavi che dovrebbero rimanere identiche all'origine inglese. Utilizzato da `validate_translation.py` per evitare avvisi di falsi positivi "non tradotti".```json
{
"description": "Keys that should remain untranslated...",
"keys": [
"common.model",
"common.oauth",
"health.cpu",
...
]
}

````

**Cosa appartiene qui:**

- Nomi di marchi/prodotti: `landing.brandName`, `common.social-github`
- Termini tecnici/acronimi: `health.cpu`, `mcpDashboard.pid`, `settings.ai`
- Stringhe ICU/formato: `apiManager.modelsCount`, `health.millisecondsShort`
- Valori segnaposto: `provviders.openaiBaseUrlPlaceholder`, `cliTools.baseUrlPlaceholder`
- Nomi dei protocolli: `common.http`, `common.oauth`, `provviders.oauth2Label`
- Sezioni di navigazione: `sidebar.primarySection`, `sidebar.cliSection`

**Per aggiungere una chiave:**Modifica l'array `keys` in `scripts/i18n/untranslatable-keys.json` ed esegui nuovamente la convalida.## CI Integration

## Best Practices

### When Editing Translations

1.**Modifica sempre prima `en.json`**: è la fonte della verità
2.**Esegui `generate-multilang.mjs message`**per propagare le nuove chiavi a tutte le versioni locali
3.**Rivedi le traduzioni automatiche**: Google Translate è un punto di partenza, non finale
4.**Convalida prima del commit**— `python3 scripts/validate_translation.py quick -l <lang>`
5.**Aggiorna `untranslatable-keys.json`**se una chiave deve rimanere in inglese### Placeholder Safety

- I segnaposto ICU (`{count}`, `{value}`, `{total}`, `{seconds}`) devono essere conservati esattamente
- I formati plurali (`{count, plural, one {# model} other {# models}}`) devono mantenere la struttura
- Il validatore rileva automaticamente le mancate corrispondenze dei segnaposto### Adding New Translation Keys in Code

```tsx
// Use namespaced keys
const t = useTranslations("settings");
t("cacheSettings"); // maps to settings.cacheSettings in JSON

// Run check_translations.py to verify keys exist
python3 scripts/check_translations.py --verbose
````

### RTL Considerations

- L'arabo (`ar`) e l'ebraico (`he`) sono versioni locali RTL
- Evita i CSS `left`/`right` codificati: utilizza le proprietà logiche `start`/`end`
- Il QA visivo rileva le mancate corrispondenze del layout RTL tramite "run-visual-qa.mjs".## Known Issues & History

### `in.json` → `hi.json` Fix

Il generatore originariamente utilizzava `code: "in"` (codice Google Translate deprecato) per l'hindi invece del corretto ISO 639-1 `hi`. Ciò ha creato un duplicato orfano "in.json" di "hi.json". Risolto il problema modificando `code: "in"` in `code: "hi"` in `generate-multilang.mjs` e rimuovendo il file orfano.### `docs/i18n/README.md` Is Auto-Generated
