OmniRoute prend en charge**30 langues**avec une traduction complète de l'interface utilisateur du tableau de bord, une documentation traduite et une prise en charge RTL pour l'arabe et l'hébreu.## Quick Reference

| Assurance qualité visuelle (dramaturge) | `node scripts/i18n/run-visual-qa.mjs`                                                    | ## Architecture |

### Source of Truth

-**Chaînes d'interface utilisateur** : `src/i18n/messages/en.json` (source anglaise, ~2800 clés) -**Fichiers locaux** : `src/i18n/messages/{locale}.json` (30 traductions) -**Framework** : `next-intl` avec résolution locale basée sur les cookies -**Config** : `src/i18n/config.ts` — définit les 30 paramètres régionaux, les noms de langue et les indicateurs### Runtime Flow

1. L'utilisateur sélectionne la langue → ensemble de cookies `NEXT_LOCALE`
2. `src/i18n/request.ts` résout les paramètres régionaux : cookie → en-tête `Accept-Language` → repli `en`
3. L'importation dynamique charge `messages/{locale}.json`
4. Les composants utilisent `useTranslations("namespace")` et `t("key")`### Supported Locales

### 1. Register the Locale

Modifiez `src/i18n/config.ts` :```ts
// Add to LOCALES array
"xx",
// Add to LANGUAGES array
{ code: "xx", label: "XX", name: "Language Name", flag: "🏳️" },

````

### 2. Add to Generator

Modifiez `scripts/i18n/generate-multilang.mjs` — ajoutez une entrée à `LOCALE_SPECS` :```js
{
  code: "xx",
  googleTl: "xx",
  label: "XX",
  flag: "🏳️",
  languageName: "Language Name",
  readmeName: "Language Name",
  docsName: "Language Name",
},
````

### 3. Generate Initial Translation

```bash
node scripts/i18n/generate-multilang.mjs messages
```

Cela crée `src/i18n/messages/xx.json` traduit automatiquement à partir de `en.json` via Google Translate.### 4. Review & Fix Auto-Translations

Les traductions automatiques sont un point de départ. Vérifiez manuellement :

- Précision technique
- Terminologie adaptée au contexte
- Gestion appropriée des espaces réservés (`{count}`, `{value}`, etc.)### 5. Validate

```bash
python3 scripts/validate_translation.py quick -l xx
python3 scripts/validate_translation.py diff common -l xx
```

### 6. Generate Translated Documentation

```bash
node scripts/i18n/generate-multilang.mjs docs
```

## Auto-Translation Pipeline

### generate-multilang.mjs (Google Translate)

**Moteur de traduction automatique principal** : utilise l'API gratuite de Google Translate pour générer des traductions pour les chaînes de l'interface utilisateur, les README et la documentation.```bash
node scripts/i18n/generate-multilang.mjs [messages|readme|docs|all]

````

| Mode | Ce qu'il fait |
| ---------- | ----------------------------------------------------------------------------- |
| `messages` | Traduit les clés manquantes dans `src/i18n/messages/{locale}.json` à partir de `en.json` |
| `lisez-moi` | Traduit `README.md` dans tous les paramètres régionaux sous la forme `README.{code}.md` à la racine du projet |
| `documents` | Traduit `DOC_SOURCE_FILES` en `docs/i18n/{locale}/{docName}` |
| `tous` | Exécute les trois modes |

**Caractéristiques :**

-**Protection du texte** : masque les blocs de code (` ``` `), le code en ligne (`` ` ``), les liens/images de démarque (`[text](url)`), les balises HTML, les tableaux et les espaces réservés ICU (`{count}`, `{value}`, `{total}`, etc.) avant la traduction, puis les restaure
-**Chunked batching** : joint plusieurs chaînes avec les délimiteurs `__OMNIROUTE_I18N_SEPARATOR__` pour minimiser les appels d'API (max 1 800 caractères par requête)
-**Cache en mémoire** : évite les appels d'API redondants pour les chaînes répétées au sein d'une session
-**Logique de nouvelle tentative** : interruption exponentielle (jusqu'à 5 tentatives avec un délai de tentative de 300 ms) pour les erreurs 429/5xx
-**Timeout** : 20 secondes par requête
-**Ignorer l'existant** : Si le fichier cible existe déjà, il n'est PAS écrasé

**Comportements importants :**

- `docs/i18n/README.md` est**régénéré**à chaque exécution — c'est un index généré automatiquement de tous les documents
- Les fichiers racine `README.{code}.md` ne sont créés que s'ils n'existent pas (ignore les paramètres régionaux dans `EXISTING_README_CODES`)
- Les barres de langue (`🌐**Langues :**...`) sont automatiquement insérées/mises à jour dans tous les documents traduits### i18n_autotranslate.py (LLM-based)

**Traducteur secondaire**— utilise n'importe quelle API LLM compatible OpenAI (y compris OmniRoute lui-même) pour traduire les fichiers de démarques `docs/i18n/` existants. Idéal pour peaufiner ou retraduire des documents avec une meilleure qualité que Google Translate.```bash
python3 scripts/i18n_autotranslate.py \
  --api-url http://localhost:20128/v1 \
  --api-key sk-your-key \
  --model gpt-4o
````

**Caractéristiques :**

- Analyse les fichiers de démarque `docs/i18n/` pour les paragraphes anglais
- Ignore les blocs de code, les tableaux et le contenu déjà traduit
- Envoie des paragraphes à LLM avec l'invite du système de traduction technique
- Prend en charge les 30 langues## Validation & QA

### validate_translation.py

**Validateur de traduction**— compare n'importe quel JSON local à « en.json » et signale les problèmes.```bash

# Quick check (counts only)

python3 scripts/validate_translation.py quick -l cs

# Output:

# Missing: 0

# Untranslated: 0

# Ignored (UNTRANSLATABLE_KEYS): 236

# Detailed diff by category

python3 scripts/validate_translation.py diff common -l cs
python3 scripts/validate_translation.py diff settings -l cs

# Export to CSV

python3 scripts/validate_translation.py csv -l cs > report.csv

# Export to Markdown

python3 scripts/validate_translation.py md -l cs > report.md

# Full report (default)

python3 scripts/validate_translation.py -l cs

````

**Détecte :**

-**Clés manquantes**— clés dans `en.json` mais pas dans le fichier de paramètres régionaux
-**Clés supplémentaires**— clés dans le fichier de paramètres régionaux mais pas dans `en.json`
-**Clés non traduites** : clés dont la valeur locale est égale à la source anglaise (hors liste autorisée)
-**Incompatibilités d'espaces réservés**— Les espaces réservés ICU qui ne correspondent pas entre la source et la traduction

**Codes de sortie :**
| Codes | Signification |
|------|--------------|
| 0 | D'accord |
| 1 | Erreur générique |
| 2 | Chaînes manquantes (erreur matérielle) |
| 3 | Avertissement non traduit (doux) |

**Environnement :**Définissez `TRANSLATION_LANG=cs` ou utilisez l'indicateur `-l cs`.### check_translations.py

**Vérificateur de clé code vers JSON**— analyse `src/**/*.tsx` et `src/**/*.ts` pour les appels `useTranslations()` et vérifie que toutes les clés référencées existent dans `en.json`.```bash
# Basic check
python3 scripts/check_translations.py

# Verbose output
python3 scripts/check_translations.py --verbose

# Auto-fix (adds missing keys to en.json)
python3 scripts/check_translations.py --fix
````

### generate-qa-checklist.mjs

**AQ d'analyse statique** : analyse les fichiers de page Next.js à la recherche de mesures de risque i18n et génère un rapport Markdown.```bash
node scripts/i18n/generate-qa-checklist.mjs

````

**Chèques :**

- Utilisation de classes à largeur fixe (risque de débordement)
- Classes directionnelles gauche/droite (risque RTL)
- Modèles sujets à l'écrêtage
- Parité locale (clés manquantes/supplémentaires vs `en.json`)
- Barres de sélection de langue README dans les locales prioritaires (`es`, `fr`, `de`, `ja`, `ar`)

**Sortie :**`docs/reports/i18n-qa-checklist-{date}.md`### run-visual-qa.mjs

** AQ visuel via Playwright ** : prend des captures d'écran de tous les itinéraires du tableau de bord dans plusieurs paramètres régionaux et fenêtres, puis évalue l'état de la page.```bash
# Default: es, fr, de, ja, ar on localhost:20128
node scripts/i18n/run-visual-qa.mjs

# Custom base URL and locales
QA_BASE_URL=http://staging.example.com QA_LOCALES=de,fr node scripts/i18n/run-visual-qa.mjs

# Custom routes
QA_ROUTES=/dashboard/settings,/dashboard/providers node scripts/i18n/run-visual-qa.mjs
````

**Détecte :**

- Débordement de texte
- Découpage d'éléments
- Incompatibilités de mise en page RTL

**Sortie :**`docs/reports/i18n-visual-qa-{date}.md` + rapport JSON## Managing Untranslatable Keys

### untranslatable-keys.json

**Fichier :**`scripts/i18n/untranslatable-keys.json`

Liste verte de clés qui doivent rester identiques à la source anglaise. Utilisé par `validate_translation.py` pour éviter les avertissements faussement positifs « non traduits ».```json
{
"description": "Keys that should remain untranslated...",
"keys": [
"common.model",
"common.oauth",
"health.cpu",
...
]
}

````

**Ce qui appartient ici :**

- Noms de marque/produit : `landing.brandName`, `common.social-github`
- Termes/acronymes techniques : `health.cpu`, `mcpDashboard.pid`, `settings.ai`
- Chaînes ICU/format : `apiManager.modelsCount`, `health.millisecondsShort`
- Valeurs d'espace réservé : `providers.openaiBaseUrlPlaceholder`, `cliTools.baseUrlPlaceholder`
- Noms des protocoles : `common.http`, `common.oauth`, `providers.oauth2Label`
- Sections de navigation : `sidebar.primarySection`, `sidebar.cliSection`

**Pour ajouter une clé :**Modifiez le tableau `keys` dans `scripts/i18n/untranslatable-keys.json` et réexécutez la validation.## CI Integration

### GitHub Actions (`.github/workflows/ci.yml`)

Le pipeline CI valide tous les paramètres régionaux à chaque push et PR :

1.**Tâche `i18n-matrix`**— découvre dynamiquement tous les fichiers de paramètres régionaux (à l'exclusion de `en.json`)
2.**`i18n` job**— exécute `validate_translation.py quick -l '<lang>'` pour chaque locale en parallèle
3.**Tâche `ci-summary`**— regroupe les résultats dans un résumé du tableau de bord```yaml

# i18n: validates each language
python3 scripts/validate_translation.py quick -l '${{ matrix.lang }}'
````

## Best Practices

### When Editing Translations

1.**Toujours modifier `en.json` en premier**— c'est la source de la vérité
2.**Exécutez `generate-multilang.mjs messages`**pour propager les nouvelles clés à tous les paramètres régionaux
3.**Réviser les traductions automatiques**— Google Translate est un point de départ et non une finalité
4.**Valider avant de s'engager**— `python3 scripts/validate_translation.py quick -l <lang>`
5.**Mettez à jour `untranslatable-keys.json`**si une clé doit rester en anglais### Placeholder Safety

- Les espaces réservés ICU (`{count}`, `{value}`, `{total}`, `{seconds}`) doivent être conservés exactement
- Les formats pluriels (`{count, plural, one {# model} other {# models}}`) doivent conserver la structure
- Le validateur détecte automatiquement les incompatibilités d'espaces réservés### Adding New Translation Keys in Code

```tsx
// Use namespaced keys
const t = useTranslations("settings");
t("cacheSettings"); // maps to settings.cacheSettings in JSON

// Run check_translations.py to verify keys exist
python3 scripts/check_translations.py --verbose
````

### RTL Considerations

- L'arabe (`ar`) et l'hébreu (`he`) sont des paramètres régionaux RTL
- Évitez les CSS `left`/`right` codés en dur — utilisez les propriétés logiques `start`/`end`
- Visual QA détecte les inadéquations de mise en page RTL via `run-visual-qa.mjs`## Known Issues & History

### `in.json` → `hi.json` Fix

Le générateur utilisait à l'origine le « code : « in » » (code Google Translate obsolète) pour l'hindi au lieu du « hi » ISO 639-1 correct. Cela a créé un double orphelin « in.json » de « hi.json ». Corrigé en changeant `code : "in"` en `code : "hi"` dans `generate-multilang.mjs` et en supprimant le fichier orphelin.### `docs/i18n/README.md` Is Auto-Generated
