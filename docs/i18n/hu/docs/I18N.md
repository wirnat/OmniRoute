### Source of Truth

-**UI karakterláncok**: `src/i18n/messages/en.json` (angol forrás, ~2800 kulcs) -**Helyi fájlok**: `src/i18n/messages/{locale}.json` (30 fordítás) -**Keretrendszer**: "next-intl" cookie-alapú területi felbontással -**Config**: `src/i18n/config.ts` – meghatározza mind a 30 területi beállítást, nyelvnevet, zászlót### Runtime Flow

1. A felhasználó kiválasztja a nyelvet → `NEXT_LOCALE` cookie set
2. Az `src/i18n/request.ts` feloldja a területi beállításokat: cookie → `Accept-Language` fejléc → tartalék `en`
3. A dinamikus import betölti az "messages/{locale}.json" fájlt
4. Az összetevők a `useTranslations("namespace")` és a `t("key")` karakterláncot használják### Supported Locales

### 1. Register the Locale

`src/i18n/config.ts` szerkesztése:```ts
// Add to LOCALES array
"xx",
// Add to LANGUAGES array
{ code: "xx", label: "XX", name: "Language Name", flag: "🏳️" },

````

### 2. Add to Generator

A `scripts/i18n/generate-multilang.mjs` szerkesztése – bejegyzés hozzáadása a `LOCALE_SPECS'-hez:```js
{
  code: "xx",
  googleTl: "xx",
  label: "XX",
  flag: "🏳️",
  languageName: "Language Name",
  readmeName: "Language Name",
  docsName: "Language Name",
},
````

### 3. Generate Initial Translation

```bash
node scripts/i18n/generate-multilang.mjs messages
```

Ez létrehozza az "src/i18n/messages/xx.json" fájlt, amelyet automatikusan lefordítanak az "en.json" fájlból a Google Fordítón keresztül.### 4. Review & Fix Auto-Translations

Az automatikus fordítás a kiindulópont. Manuális ellenőrzés:

- Műszaki pontosság
- Kontextusnak megfelelő terminológia
- A helyőrzők megfelelő kezelése (`{count}`, `{value}` stb.)### 5. Validate

```bash
python3 scripts/validate_translation.py quick -l xx
python3 scripts/validate_translation.py diff common -l xx
```

### 6. Generate Translated Documentation

```bash
node scripts/i18n/generate-multilang.mjs docs
```

## Auto-Translation Pipeline

### generate-multilang.mjs (Google Translate)

**Elsődleges automatikus fordítómotor**– a Google Fordító ingyenes API-ját használja a felhasználói felületi karakterláncok, README-k és dokumentáció fordításainak előállításához.```bash
node scripts/i18n/generate-multilang.mjs [messages|readme|docs|all]

````

| mód | Mit csinál |
| ---------- | ----------------------------------------------------------------------------- |
| "üzenetek" | Lefordítja a hiányzó kulcsokat a `src/i18n/messages/{locale}.json` fájlból az `en.json` fájlból |
| "olvass el" | Lefordítja a `README.md` fájlt minden területre `README.{code}.md` néven a projekt gyökérben |
| "dokumentumok" | Lefordítja a `DOC_SOURCE_FILES` fájlt `docs/i18n/{locale}/{docName}' nyelvre |
| `mind` | Mindhárom módot futtatja |

**Jellemzők:**

-**Szövegvédelem**: Maszkolja a kódblokkokat (` ``` `), a soron belüli kódot (`` ` ``), a leíró hivatkozásokat/képeket (`[szöveg](url)`), a HTML-címkéket, a táblázatokat és az ICU-helyőrzőket (`{count}`, `{value}`, `{total}` stb.) a fordítás előtt, majd visszaállítja őket.
-**Részesített kötegelés**: Több karakterláncot egyesít `__OMNIROUTE_I18N_SEPARATOR__` elválasztójelekkel az API-hívások minimalizálása érdekében (kérésenként legfeljebb 1800 karakter)
-**A memórián belüli gyorsítótár**: Megakadályozza a redundáns API-hívásokat ismétlődő karakterláncokhoz egy munkameneten belül
-**Újrapróbálkozási logika**: Exponenciális visszalépés (legfeljebb 5 kísérlet 300 ms × kísérleti késleltetéssel) 429/5xx hibákhoz
-**Időtúllépés**: kérésenként 20 másodperc
-**Létező átugrása**: Ha már létezik célfájl, az NEM kerül felülírásra

**Fontos viselkedések:**

- A `docs/i18n/README.md` minden futtatáskor**újragenerálódik**– ez az összes dokumentum automatikusan generált indexe
- A gyökér `README.{code}.md` fájlok csak akkor jönnek létre, ha nem léteznek (kihagyja a nyelvi beállításokat a `EXISTING_README_CODES'-ban)
- A nyelvi sávok (`🌐**Nyelvek:**...`) automatikusan beszúródnak/frissülnek minden lefordított dokumentumba### i18n_autotranslate.py (LLM-based)

**Másodlagos fordító**– bármely OpenAI-kompatibilis LLM API-t használ (beleértve magát az OmniRoute-ot is) a meglévő "docs/i18n/" leíró fájlok lefordításához. A legjobb a dokumentumok csiszolásához vagy újrafordításához, jobb minőségben, mint a Google Fordító.```bash
python3 scripts/i18n_autotranslate.py \
  --api-url http://localhost:20128/v1 \
  --api-key sk-your-key \
  --model gpt-4o
````

**Jellemzők:**

- Ellenőrzi a `docs/i18n/` lejelölő fájlokat az angol bekezdésekért
- Kihagyja a kódblokkokat, táblázatokat és a már lefordított tartalmakat
- Bekezdéseket küld az LLM-nek a műszaki fordítási rendszer felszólításával
- Támogatja mind a 30 nyelvet## Validation & QA

### validate_translation.py

**Fordítás-ellenőrző**– összehasonlítja a nyelvi JSON-okat az "en.json"-val, és jelenti a problémákat.```bash

# Quick check (counts only)

python3 scripts/validate_translation.py quick -l cs

# Output:

# Missing: 0

# Untranslated: 0

# Ignored (UNTRANSLATABLE_KEYS): 236

# Detailed diff by category

python3 scripts/validate_translation.py diff common -l cs
python3 scripts/validate_translation.py diff settings -l cs

# Export to CSV

python3 scripts/validate_translation.py csv -l cs > report.csv

# Export to Markdown

python3 scripts/validate_translation.py md -l cs > report.md

# Full report (default)

python3 scripts/validate_translation.py -l cs

````

**Érzékeli:**

-**Hiányzó kulcsok**– kulcsok az `en.json-ban, de nem a területi beállításfájlban
-**Extra kulcsok**– kulcsok a területi beállításfájlban, de nem az „en.json” fájlban
-**Nem lefordított kulcsok**– olyan kulcsok, ahol a nyelvi érték megegyezik az angol forrással (kivéve az engedélyezési listát)
-**Helyőrző eltérések**— ICU-helyőrzők, amelyek nem egyeznek a forrás és a fordítás között

**Kilépési kódok:**
| Kód | Jelentése |
|------|---------|
| 0 | OK |
| 1 | Általános hiba |
| 2 | Hiányzó karakterláncok (kemény hiba) |
| 3 | Lefordítatlan figyelmeztetés (puha) |

**Környezet:**Állítsa be a „TRANSLATION_LANG=cs” értéket, vagy használja a „-l cs” jelzőt.### check_translations.py

**Kód-JSON-kulcsellenőrző**– megvizsgálja az „src/**/*.tsx” és az „src/**/*.ts” függvényt a „useTranslations()” hívásokra, és ellenőrzi, hogy minden hivatkozott kulcs létezik az „en.json” fájlban.```bash
# Basic check
python3 scripts/check_translations.py

# Verbose output
python3 scripts/check_translations.py --verbose

# Auto-fix (adds missing keys to en.json)
python3 scripts/check_translations.py --fix
````

### untranslatable-keys.json

**Fájl:**`scripts/i18n/untranslable-keys.json`

Azon kulcsok engedélyezési listája, amelyeknek meg kell egyezniük az angol forráskóddal. A `validate_translation.py` használja a hamis pozitív "nem lefordított" figyelmeztetések elkerülésére.```json
{
"description": "Keys that should remain untranslated...",
"keys": [
"common.model",
"common.oauth",
"health.cpu",
...
]
}

````

**Mi tartozik ide:**

- Márka-/terméknevek: "landing.brandName", "common.social-github"
- Szakkifejezések/mozaikszavak: "health.cpu", "mcpDashboard.pid", "settings.ai"
- ICU/formátum karakterláncok: "apiManager.modelsCount", "health.millisecondsShort"
- Helyőrző értékek: `providers.openaiBaseUrlPlaceholder`, `cliTools.baseUrlPlaceholder`
- Protokollnevek: "common.http", "common.oauth", "providers.oauth2Label"
- Navigációs szakaszok: "sidebar.primarySection", "sidebar.cliSection"

**Kulcs hozzáadása:**Szerkessze a "keys" tömböt a scripts/i18n/untranslable-keys.json fájlban, és futtassa újra az érvényesítést.## CI Integration

## Best Practices

### When Editing Translations

1.**Először mindig az „en.json” fájlt szerkessze**– ez az igazság forrása
2.**Futtassa a `generate-multilang.mjs messages`**parancsot az új kulcsok terjesztéséhez az összes területi beállításhoz.
3.**Tekintse át az automatikus fordításokat**– A Google Fordító nem végleges, hanem kiindulópont
4.**Elvégzés előtti ellenőrzés**— `python3 scripts/validate_translation.py quick -l <lang>`
5.**Frissítse az `untranslable-keys.json`**fájlt, ha a kulcs angol nyelvű marad.### Placeholder Safety

- Az ICU-helyőrzőket ("{count}", "{value}", "{total}", "{seconds}") pontosan meg kell őrizni
- A többes számú formátumoknak (`{count, plural, one {# model} other {# model}}`) meg kell őrizniük a szerkezetet
- Az érvényesítő automatikusan észleli a helyőrző eltéréseket### Adding New Translation Keys in Code

```tsx
// Use namespaced keys
const t = useTranslations("settings");
t("cacheSettings"); // maps to settings.cacheSettings in JSON

// Run check_translations.py to verify keys exist
python3 scripts/check_translations.py --verbose
````

### RTL Considerations

- Az arab ("ar") és a héber ("he") az RTL területei
- Kerülje a keménykódolt "bal"/"jobb" CSS-t – használja a "start"/"end" logikai tulajdonságokat
- A vizuális minőségbiztosítás a "run-visual-qa.mjs" segítségével észleli az RTL elrendezés eltéréseit## Known Issues & History

### `in.json` → `hi.json` Fix

A generátor eredetileg a "code: "in"" kódot (elavult Google Fordító kód) használta hindi nyelven a helyes ISO 639-1 "hi" helyett. Ezzel a „hi.json” egy árva „in.json” másolata jött létre. Javítva: a `code: "in"" a `code: "hi"-re változott a `generate-multilang.mjs` fájlban, és eltávolította az árva fájlt.### `docs/i18n/README.md` Is Auto-Generated

A "docs/i18n/README.md" fájlt a "generate-multilang.mjs docs" teljesen újragenerálja. A manuális szerkesztések elvesznek. Használja a "docs/I18N.md" fájlt (ezt a fájlt) a kézzel írt dokumentációhoz, amely továbbra is fennáll.### External Untranslatable Keys List

Az "untranslable-keys.json" engedélyezési lista a "validate_translation.py" fájlban található beépített Python-készletből egy külső JSON-fájlba került a könnyebb karbantartás érdekében. A validátor futás közben tölti be.### `generate-multilang.mjs` Hindi Code Fix
