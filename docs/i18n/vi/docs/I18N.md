### Source of Truth

-**Chuỗi giao diện người dùng**: `src/i18n/messages/en.json` (nguồn tiếng Anh, ~2800 phím) -**Tệp ngôn ngữ**: `src/i18n/messages/{locale}.json` (30 bản dịch) -**Khung**: `next-intl` với độ phân giải ngôn ngữ dựa trên cookie -**Config**: `src/i18n/config.ts` — xác định tất cả 30 ngôn ngữ, tên ngôn ngữ, cờ### Runtime Flow

1. Người dùng chọn ngôn ngữ → bộ cookie `NEXT_LOCALE`
2. `src/i18n/request.ts` phân giải ngôn ngữ: cookie → tiêu đề `Accept-Language` → dự phòng `en`
3. Tải động nhập `messages/{locale}.json`
4. Các thành phần sử dụng `useTranslations("namespace")` và `t("key")`### Supported Locales

### 1. Register the Locale

Chỉnh sửa `src/i18n/config.ts`:```ts
// Add to LOCALES array
"xx",
// Add to LANGUAGES array
{ code: "xx", label: "XX", name: "Language Name", flag: "🏳️" },

````

### 2. Add to Generator

Chỉnh sửa `scripts/i18n/generate-multilang.mjs` — thêm mục nhập vào `LOCALE_SPECS`:```js
{
  code: "xx",
  googleTl: "xx",
  label: "XX",
  flag: "🏳️",
  languageName: "Language Name",
  readmeName: "Language Name",
  docsName: "Language Name",
},
````

### 3. Generate Initial Translation

```bash
node scripts/i18n/generate-multilang.mjs messages
```

Điều này tạo ra `src/i18n/messages/xx.json` được dịch tự động từ `en.json` thông qua Google Dịch.### 4. Review & Fix Auto-Translations

Tự động dịch là điểm khởi đầu. Xem xét thủ công cho:

- Độ chính xác kỹ thuật
- Thuật ngữ phù hợp với ngữ cảnh
- Xử lý đúng cách các phần giữ chỗ (`{count}`, `{value}`, v.v.)### 5. Validate

```bash
python3 scripts/validate_translation.py quick -l xx
python3 scripts/validate_translation.py diff common -l xx
```

### 6. Generate Translated Documentation

```bash
node scripts/i18n/generate-multilang.mjs docs
```

## Auto-Translation Pipeline

### generate-multilang.mjs (Google Translate)

**Công cụ dịch tự động chính**— sử dụng API miễn phí của Google Translate để tạo bản dịch cho chuỗi giao diện người dùng, README và tài liệu.```bash
node scripts/i18n/generate-multilang.mjs [messages|readme|docs|all]

````

| Chế độ | Nó làm gì |
| ---------- | ----------------------------------------------------------------------------- |
| `tin nhắn` | Dịch các khóa bị thiếu trong `src/i18n/messages/{locale}.json` từ `en.json` |
| `đọc tôi` | Dịch `README.md` sang tất cả các ngôn ngữ dưới dạng `README.{code}.md` trong thư mục gốc của dự án |
| `tài liệu` | Dịch `DOC_SOURCE_FILES` sang `docs/i18n/{locale}/{docName}` |
| `tất cả` | Chạy cả ba chế độ |

**Tính năng:**

-**Bảo vệ văn bản**: Che các khối mã (` ``` `), mã nội tuyến (`` ` ``), liên kết/hình ảnh đánh dấu (`[text](url)`), thẻ HTML, bảng và phần giữ chỗ ICU (`{count}`, `{value}`, `{total}`, v.v.) trước khi dịch, sau đó khôi phục chúng
-**Phân nhóm theo khối**: Nối nhiều chuỗi bằng dấu phân cách `__OMNIROUTE_I18N_SEPARATOR__` để giảm thiểu lệnh gọi API (tối đa 1800 ký tự cho mỗi yêu cầu)
-**Bộ nhớ đệm trong bộ nhớ**: Tránh các lệnh gọi API dư thừa cho các chuỗi lặp lại trong một phiên
-**Logic thử lại**: Độ trễ theo cấp số nhân (tối đa 5 lần thử với 300 mili giây × độ trễ lần thử) đối với lỗi 429/5xx
-**Thời gian chờ**: 20 giây cho mỗi yêu cầu
-**Bỏ qua hiện có**: Nếu file đích đã tồn tại thì KHÔNG bị ghi đè

**Những hành vi quan trọng:**

- `docs/i18n/README.md` được**tái tạo**mỗi lần chạy — đó là chỉ mục được tạo tự động của tất cả tài liệu
- Tệp gốc `README.{code}.md` chỉ được tạo nếu chúng không tồn tại (bỏ qua các ngôn ngữ trong `EXISTING_README_CODES`)
- Thanh ngôn ngữ (`🌐**Ngôn ngữ:**...`) được tự động chèn/cập nhật trong tất cả các tài liệu đã dịch### i18n_autotranslate.py (LLM-based)

**Trình dịch phụ**— sử dụng mọi API LLM tương thích với OpenAI (bao gồm cả OmniRoute) để dịch các tệp đánh dấu `docs/i18n/` hiện có. Tốt nhất để đánh bóng hoặc dịch lại tài liệu với chất lượng tốt hơn Google Dịch.```bash
python3 scripts/i18n_autotranslate.py \
  --api-url http://localhost:20128/v1 \
  --api-key sk-your-key \
  --model gpt-4o
````

**Tính năng:**

- Quét các tập tin đánh dấu `docs/i18n/` cho các đoạn văn tiếng Anh
- Bỏ qua các khối mã, bảng và nội dung đã được dịch
- Gửi đoạn văn tới LLM với lời nhắc của hệ thống dịch thuật kỹ thuật
- Hỗ trợ tất cả 30 ngôn ngữ## Validation & QA

### validate_translation.py

**Trình xác thực bản dịch**— so sánh bất kỳ JSON ngôn ngữ nào với `en.json` và báo cáo vấn đề.```bash

# Quick check (counts only)

python3 scripts/validate_translation.py quick -l cs

# Output:

# Missing: 0

# Untranslated: 0

# Ignored (UNTRANSLATABLE_KEYS): 236

# Detailed diff by category

python3 scripts/validate_translation.py diff common -l cs
python3 scripts/validate_translation.py diff settings -l cs

# Export to CSV

python3 scripts/validate_translation.py csv -l cs > report.csv

# Export to Markdown

python3 scripts/validate_translation.py md -l cs > report.md

# Full report (default)

python3 scripts/validate_translation.py -l cs

````

**Phát hiện:**

-**Thiếu khóa**— khóa trong `en.json` nhưng không có trong tệp ngôn ngữ
-**Khóa bổ sung**— khóa trong tệp ngôn ngữ nhưng không có trong `en.json`
-**Khóa chưa được dịch**— khóa trong đó giá trị ngôn ngữ bằng nguồn tiếng Anh (không bao gồm danh sách cho phép)
-**Phần giữ chỗ không khớp**— Phần giữ chỗ ICU không khớp giữa nguồn và bản dịch

**Mã thoát:**
| Mã | Ý nghĩa |
|------|----------|
| 0 | được |
| 1 | Lỗi chung |
| 2 | Thiếu chuỗi (lỗi cứng) |
| 3 | Cảnh báo chưa được dịch (mềm) |

**Môi trường:**Đặt `TRANSLATION_LANG=cs` hoặc sử dụng cờ `-l cs`.### check_translations.py

**Trình kiểm tra khóa Code-to-JSON**— quét `src/**/*.tsx` và `src/**/*.ts` để tìm lệnh gọi `useTranslations()` và xác minh tất cả các khóa được tham chiếu tồn tại trong `en.json`.```bash
# Basic check
python3 scripts/check_translations.py

# Verbose output
python3 scripts/check_translations.py --verbose

# Auto-fix (adds missing keys to en.json)
python3 scripts/check_translations.py --fix
````

### untranslatable-keys.json

**Tệp:**`scripts/i18n/untranslatable-keys.json`

Danh sách cho phép các khóa phải giống với nguồn tiếng Anh. Được sử dụng bởi `validate_translation.py` để tránh cảnh báo "chưa được dịch" dương tính giả.```json
{
"description": "Keys that should remain untranslated...",
"keys": [
"common.model",
"common.oauth",
"health.cpu",
...
]
}

````

**Những gì thuộc về nơi này:**

- Tên thương hiệu/sản phẩm: `landing.brandName`, `common.social-github`
- Thuật ngữ/từ viết tắt kỹ thuật: `health.cpu`, `mcpDashboard.pid`, `settings.ai`
- Chuỗi định dạng/ICU: `apiManager.modelsCount`, `health.millisecondsShort`
- Giá trị giữ chỗ: `providers.openaiBaseUrlPlaceholder`, `cliTools.baseUrlPlaceholder`
- Tên giao thức: `common.http`, `common.oauth`, `providers.oauth2Label`
- Các phần điều hướng: `sidebar.primarySection`, `sidebar.cliSection`

**Cách thêm khóa:**Chỉnh sửa mảng `keys` trong `scripts/i18n/untranslatable-keys.json` và chạy lại quá trình xác thực.## CI Integration

## Best Practices

### When Editing Translations

1.**Luôn chỉnh sửa `en.json` trước**— đó là nguồn gốc của sự thật
2.**Chạy `generate-multilang.mjs messages`**để truyền các khóa mới tới tất cả các ngôn ngữ
3.**Xem lại bản dịch tự động**— Google Dịch là điểm khởi đầu, không phải là điểm cuối cùng
4.**Xác thực trước khi cam kết**— `python3 scripts/validate_translation.py quick -l <lang>`
5.**Cập nhật `untranslatable-keys.json`**nếu khóa vẫn bằng tiếng Anh### Placeholder Safety

- Phần giữ chỗ ICU (`{count}`, `{value}`, `{total}`, `{seconds}`) phải được giữ nguyên chính xác
- Định dạng số nhiều (`{đếm, số nhiều, một {# mô hình} khác {# mô hình}}`) phải duy trì cấu trúc
- Trình xác thực tự động phát hiện phần giữ chỗ không khớp### Adding New Translation Keys in Code

```tsx
// Use namespaced keys
const t = useTranslations("settings");
t("cacheSettings"); // maps to settings.cacheSettings in JSON

// Run check_translations.py to verify keys exist
python3 scripts/check_translations.py --verbose
````

### RTL Considerations

- Tiếng Ả Rập (`ar`) và tiếng Do Thái (`he`) là ngôn ngữ RTL
- Tránh CSS `left`/`right` được mã hóa cứng - sử dụng các thuộc tính logic `start`/`end`
- Visual QA phát hiện bố cục RTL không khớp thông qua `run-visual-qa.mjs`## Known Issues & History

### `in.json` → `hi.json` Fix

Trình tạo ban đầu sử dụng `code: "in"` (mã Google Dịch không được dùng nữa) cho tiếng Hindi thay vì ISO 639-1 `hi` chính xác. Điều này đã tạo ra một bản sao `in.json` mồ côi của `hi.json`. Đã sửa bằng cách thay đổi `code: "in"` thành `code: "hi"` trong `generate-multilang.mjs` và xóa tệp mồ côi.### `docs/i18n/README.md` Is Auto-Generated

Tệp `docs/i18n/README.md` được tái tạo hoàn toàn bởi `generate-multilang.mjs docs`. Mọi chỉnh sửa thủ công sẽ bị mất. Sử dụng `docs/I18N.md` (tệp này) cho tài liệu viết tay vẫn tồn tại.### External Untranslatable Keys List
