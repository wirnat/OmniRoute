### Source of Truth

-**Строки пользовательского интерфейса**: `src/i18n/messages/en.json` (исходный код на английском языке, ~2800 ключей) -**Файлы локали**: `src/i18n/messages/{locale}.json` (30 переводов) -**Framework**: `next-intl` с разрешением локали на основе файлов cookie. -**Config**: `src/i18n/config.ts` — определяет все 30 локалей, названия языков, флаги.### Runtime Flow

1. Пользователь выбирает язык → набор файлов cookie `NEXT_LOCALE`.
2. `src/i18n/request.ts` разрешает локаль: cookie → заголовок `Accept-Language` → резервный `en`
3. Динамический импорт загружает `messages/{locale}.json`
4. Компоненты используют `useTranslations("namespace")` и `t("key")`### Supported Locales

### 1. Register the Locale

Отредактируйте `src/i18n/config.ts`:```ts
// Add to LOCALES array
"xx",
// Add to LANGUAGES array
{ code: "xx", label: "XX", name: "Language Name", flag: "🏳️" },

````

### 2. Add to Generator

Отредактируйте `scripts/i18n/generate-multilang.mjs` — добавьте запись в `LOCALE_SPECS`:```js
{
  code: "xx",
  googleTl: "xx",
  label: "XX",
  flag: "🏳️",
  languageName: "Language Name",
  readmeName: "Language Name",
  docsName: "Language Name",
},
````

### 3. Generate Initial Translation

```bash
node scripts/i18n/generate-multilang.mjs messages
```

При этом создается `src/i18n/messages/xx.json`, автоматически переведенный из `en.json` через Google Translate.### 4. Review & Fix Auto-Translations

Автопереводы — это отправная точка. Проверьте вручную:

- Техническая точность
- Терминология, соответствующая контексту
- Правильная обработка заполнителей (`{count}`, `{value}` и т. д.)### 5. Validate

```bash
python3 scripts/validate_translation.py quick -l xx
python3 scripts/validate_translation.py diff common -l xx
```

### 6. Generate Translated Documentation

```bash
node scripts/i18n/generate-multilang.mjs docs
```

## Auto-Translation Pipeline

### generate-multilang.mjs (Google Translate)

**Основной механизм автоматического перевода** — использует бесплатный API Google Translate для создания переводов строк пользовательского интерфейса, файлов README и документации.```bash
node scripts/i18n/generate-multilang.mjs [messages|readme|docs|all]

`````

| Режим | Что он делает |
| ---------- | ----------------------------------------------------------------------------- |
| `сообщения` | Переводит отсутствующие ключи в `src/i18n/messages/{locale}.json` из `en.json` |
| `читать` | Переводит `README.md` на все локали как `README.{code}.md` в корне проекта |
| `документы` | Переводит `DOC_SOURCE_FILES` в `docs/i18n/{locale}/{docName}` |
| `все` | Работает во всех трех режимах |

**Особенности:**

-**Защита текста**: маскирует блоки кода (` ````), встроенный код (`` ```), ссылки/изображения уценки (`[text](url)`), теги HTML, таблицы и заполнители ICU (`{count}`, `{value}`, `{total}` и т. д.) перед переводом, а затем восстанавливает их.
-**Частная пакетная обработка**: объединяет несколько строк с разделителями `__OMNIROUTE_I18N_SEPARATOR__` для минимизации вызовов API (максимум 1800 символов на запрос).
-**Кэш в памяти**: позволяет избежать избыточных вызовов API для повторяющихся строк в сеансе.
-**Логика повтора**: экспоненциальная задержка (до 5 попыток с задержкой 300 мс × попытка) для ошибок 429/5xx.
-**Тайм-аут**: 20 секунд на запрос.
-**Пропустить существующий**: если целевой файл уже существует, он НЕ перезаписывается.

**Важные правила поведения:**

- `docs/i18n/README.md`**перегенерируется**при каждом запуске — это автоматически создаваемый индекс всех документов.
- Корневые файлы `README.{code}.md` создаются только в том случае, если они не существуют (пропускаются локали в `EXISTING_README_CODES`)
- Языковые панели (`🌐**Языки:**...`) автоматически вставляются/обновляются во всех переведенных документах.### i18n_autotranslate.py (LLM-based)

**Вторичный переводчик**— использует любой OpenAI-совместимый API LLM (включая сам OmniRoute) для перевода существующих файлов уценки `docs/i18n/`. Лучше всего подходит для доработки или повторного перевода документов с более высоким качеством, чем Google Translate.```bash
python3 scripts/i18n_autotranslate.py \
  --api-url http://localhost:20128/v1 \
  --api-key sk-your-key \
  --model gpt-4o
`````

**Особенности:**

- Сканирует файлы `docs/i18n/` на наличие английских абзацев.
- Пропускает блоки кода, таблицы и уже переведенный контент.
- Отправляет абзацы в LLM с подсказкой системы технического перевода.
- Поддерживает все 30 языков## Validation & QA

### validate_translation.py

**Валидатор перевода** – сравнивает JSON любого языкового стандарта с `en.json` и сообщает о проблемах.```bash

# Quick check (counts only)

python3 scripts/validate_translation.py quick -l cs

# Output:

# Missing: 0

# Untranslated: 0

# Ignored (UNTRANSLATABLE_KEYS): 236

# Detailed diff by category

python3 scripts/validate_translation.py diff common -l cs
python3 scripts/validate_translation.py diff settings -l cs

# Export to CSV

python3 scripts/validate_translation.py csv -l cs > report.csv

# Export to Markdown

python3 scripts/validate_translation.py md -l cs > report.md

# Full report (default)

python3 scripts/validate_translation.py -l cs

````

**Обнаруживает:**

-**Отсутствуют ключи**— ключи в `en.json`, но отсутствуют в файле локали.
-**Дополнительные ключи**— ключи в файле локали, но не в `en.json`
-**Непереведенные ключи**— ключи, для которых значение локали соответствует английскому источнику (исключая список разрешенных).
-**Несоответствия заполнителей**— заполнители ICU, которые не совпадают между источником и переводом.

**Коды выхода:**
| Код | Значение |
|------|---------|
| 0 | ОК |
| 1 | Общая ошибка |
| 2 | Отсутствуют строки (серьезная ошибка) |
| 3 | Непереведенное предупреждение (программное) |

**Среда:**Установите `TRANSLATION_LANG=cs` или используйте флаг `-l cs`.### check_translations.py

**Проверка ключей Code-to-JSON**— сканирует `src/**/*.tsx` и `src/**/*.ts` на наличие вызовов `useTranslations()` и проверяет наличие всех указанных ключей в `en.json`.```bash
# Basic check
python3 scripts/check_translations.py

# Verbose output
python3 scripts/check_translations.py --verbose

# Auto-fix (adds missing keys to en.json)
python3 scripts/check_translations.py --fix
````

**Вывод:**`docs/reports/i18n-visual-qa-{date}.md` + отчет JSON## Managing Untranslatable Keys

### untranslatable-keys.json

**Файл:**`scripts/i18n/untranslatable-keys.json`

Разрешенный список ключей, которые должны оставаться идентичными английскому источнику. Используется `validate_translation.py`, чтобы избежать ложноположительных «непереведенных» предупреждений.```json
{
"description": "Keys that should remain untranslated...",
"keys": [
"common.model",
"common.oauth",
"health.cpu",
...
]
}

````

**Что сюда относится:**

– Названия брендов/продуктов: «landing.brandName», «common.social-github».
- Технические термины/аббревиатуры: «health.cpu», «mcpDashboard.pid», «settings.ai».
- Строки ICU/формата: `apiManager.modelsCount`, `health.milliсекундыShort`
- Значения заполнителей: `providers.openaiBaseUrlPlaceholder`, `cliTools.baseUrlPlaceholder`
- Имена протоколов: common.http, common.oauth,providers.oauth2Label.
- Разделы навигации: `sidebar.primarySection`, `sidebar.cliSection`

**Чтобы добавить ключ:**Отредактируйте массив `keys` в `scripts/i18n/untranslatable-keys.json` и повторно запустите проверку.## CI Integration

## Best Practices

### When Editing Translations

1.**Всегда сначала редактируйте `en.json`**— это источник истины.
2.**Запустите `generate-multilang.mjs messages`**, чтобы распространить новые ключи на все локали.
3.**Проверьте автоматические переводы**. Google Translate — это отправная точка, а не конечная точка.
4.**Проверка перед фиксацией**— `python3 scripts/validate_translation.py fast -l <lang>`
5.**Обновите `untranslatable-keys.json`**, если ключ должен оставаться на английском языке.### Placeholder Safety

- Заполнители ICU (`{count}`, `{value}`, `{total}`, `{секунды}`) должны сохраняться точно.
– Форматы множественного числа (`{count, множественное число, one {# модель}other {# модели}}`) должны сохранять структуру.
- Валидатор автоматически обнаруживает несоответствия заполнителей.### Adding New Translation Keys in Code

```tsx
// Use namespaced keys
const t = useTranslations("settings");
t("cacheSettings"); // maps to settings.cacheSettings in JSON

// Run check_translations.py to verify keys exist
python3 scripts/check_translations.py --verbose
````

### RTL Considerations

- Арабский (`ar`) и иврит (`he`) являются языковыми стандартами RTL.
- Избегайте жестко закодированных CSS-кодов «left»/«right» — используйте логические свойства «start» и «end».
- Visual QA обнаруживает несоответствия макета RTL с помощью run-visual-qa.mjs.## Known Issues & History

### `in.json` → `hi.json` Fix

Первоначально генератор использовал `code: "in"` (устаревший код Google Translate) для хинди вместо правильного ISO 639-1 `hi`. В результате был создан потерянный дубликат in.json файла hi.json. Исправлено путем замены `code: "in"` на `code: "hi"` в `generate-multilang.mjs` и удаления потерянного файла.### `docs/i18n/README.md` Is Auto-Generated

Файл `docs/i18n/README.md` полностью воссоздается с помощью `generate-multilang.mjs docs`. Любые изменения, внесенные вручную, будут потеряны. Используйте `docs/I18N.md` (этот файл) для рукописной документации, которая должна сохраниться.### External Untranslatable Keys List

Список разрешений `untranslatable-keys.json` был перенесен из встроенного Python, установленного в `validate_translation.py`, во внешний файл JSON для упрощения обслуживания. Валидатор загружает его во время выполнения.### `generate-multilang.mjs` Hindi Code Fix
