### Source of Truth

-**UI-strängar**: `src/i18n/messages/en.json` (engelsk källa, ~2800 nycklar) -**Locale filer**: `src/i18n/messages/{locale}.json` (30 översättningar) -**Framework**: `next-intl` med cookie-baserad lokalupplösning -**Config**: `src/i18n/config.ts` — definierar alla 30 språk, språknamn, flaggor### Runtime Flow

1. Användaren väljer språk → `NEXT_LOCALE` cookie set
2. `src/i18n/request.ts` löser språk: cookie → `Accept-Language` header → fallback `en`
3. Dynamisk import laddar `meddelanden/{locale}.json`
4. Komponenter använder `useTranslations("namespace")` och `t("key")`### Supported Locales

### 1. Register the Locale

Redigera `src/i18n/config.ts`:```ts
// Add to LOCALES array
"xx",
// Add to LANGUAGES array
{ code: "xx", label: "XX", name: "Language Name", flag: "🏳️" },

````

### 2. Add to Generator

Redigera `scripts/i18n/generate-multilang.mjs` — lägg till posten i `LOCALE_SPECS`:```js
{
  code: "xx",
  googleTl: "xx",
  label: "XX",
  flag: "🏳️",
  languageName: "Language Name",
  readmeName: "Language Name",
  docsName: "Language Name",
},
````

### 3. Generate Initial Translation

```bash
node scripts/i18n/generate-multilang.mjs messages
```

Detta skapar `src/i18n/messages/xx.json` automatiskt översatt från `en.json` via Google Translate.### 4. Review & Fix Auto-Translations

Automatiska översättningar är en utgångspunkt. Granska manuellt för:

- Teknisk noggrannhet
- Kontextanpassad terminologi
- Korrekt hantering av platshållare (`{count}`, `{value}` osv.)### 5. Validate

```bash
python3 scripts/validate_translation.py quick -l xx
python3 scripts/validate_translation.py diff common -l xx
```

### 6. Generate Translated Documentation

```bash
node scripts/i18n/generate-multilang.mjs docs
```

## Auto-Translation Pipeline

### generate-multilang.mjs (Google Translate)

**Primär motor för automatisk översättning**— använder Google Translates gratis API för att generera översättningar för UI-strängar, README:er och dokumentation.```bash
node scripts/i18n/generate-multilang.mjs [messages|readme|docs|all]

`````

| Läge | Vad det gör |
| ---------- | ------------------------------------------------------------------------------------ |
| `meddelanden` | Översätter saknade nycklar i `src/i18n/messages/{locale}.json` från `en.json` |
| "läs mig" | Översätter `README.md` till alla språk som `README.{code}.md` i projektroten |
| `dokument` | Översätter `DOC_SOURCE_FILES` till `docs/i18n/{locale}/{docName}` |
| `alla` | Kör alla tre lägen |

**Funktioner:**

-**Textskydd**: Maskerar kodblock (```` `), inline-kod (`` ```), markdown-länkar/bilder (`[text](url)`), HTML-taggar, tabeller och ICU-platshållare (`{count}`, `{value}`, `{total}`, etc.) före översättning och återställer dem sedan
-**Chunked batching**: Sammanfogar flera strängar med `__OMNIROUTE_I18N_SEPARATOR__` avgränsare för att minimera API-anrop (max 1800 tecken per begäran)
-**Cache i minnet**: Undviker redundanta API-anrop för upprepade strängar inom en session
-**Försök logik igen**: Exponentiell backoff (upp till 5 försök med 300ms × försöksfördröjning) för 429/5xx-fel
-**Timeout**: 20 sekunder per begäran
-**Hoppa över befintlig**: Om målfilen redan finns, skrivs den INTE över

**Viktigt beteende:**

- `docs/i18n/README.md`**regenereras**varje körning – det är ett automatiskt genererat index över alla dokument
- Root `README.{code}.md`-filer skapas bara om de inte finns (hoppar över språk i `EXISTING_README_CODES`)
- Språkfält (`🌐**Språk:**...`) infogas/uppdateras automatiskt i alla översatta dokument### i18n_autotranslate.py (LLM-based)

**Sekundär översättare**— använder alla OpenAI-kompatibla LLM API (inklusive OmniRoute själv) för att översätta befintliga `docs/i18n/` markdown-filer. Bäst för att polera eller omöversätta dokument med bättre kvalitet än Google Översätt.```bash
python3 scripts/i18n_autotranslate.py \
  --api-url http://localhost:20128/v1 \
  --api-key sk-your-key \
  --model gpt-4o
`````

**Funktioner:**

- Skannar "docs/i18n/" markdown-filer för engelska stycken
- Hoppa över kodblock, tabeller och redan översatt innehåll
- Skickar stycken till LLM med teknisk översättningssystemprompt
- Stöder alla 30 språk## Validation & QA

### validate_translation.py

**Översättningsvalidator**— jämför vilken JSON som helst med en.json och rapporterar problem.```bash

# Quick check (counts only)

python3 scripts/validate_translation.py quick -l cs

# Output:

# Missing: 0

# Untranslated: 0

# Ignored (UNTRANSLATABLE_KEYS): 236

# Detailed diff by category

python3 scripts/validate_translation.py diff common -l cs
python3 scripts/validate_translation.py diff settings -l cs

# Export to CSV

python3 scripts/validate_translation.py csv -l cs > report.csv

# Export to Markdown

python3 scripts/validate_translation.py md -l cs > report.md

# Full report (default)

python3 scripts/validate_translation.py -l cs

````

**Detekterar:**

-**Nycklar saknas**— nycklar i `en.json` men inte i språkfil
-**Extra nycklar**— nycklar i språkfil men inte i `en.json`
-**Oöversatta nycklar**— nycklar där språkvärdet är lika med engelsk källa (exklusive godkännandelista)
-**Platshållare missmatchar**— ICU-platshållare som inte matchar mellan källa och översättning

**Utgångskoder:**
| Kod | Betydelse |
|------|--------|
| 0 | OK |
| 1 | Generiskt fel |
| 2 | Saknade strängar (svårt fel) |
| 3 | Oöversatt varning (mjuk) |

**Miljö:**Ställ in "TRANSLATION_LANG=cs" eller använd flaggan "-l cs".### check_translations.py

**Kod-till-JSON-nyckelkontroll**— skannar `src/**/*.tsx` och `src/**/*.ts` efter `useTranslations()`-anrop och verifierar att alla refererade nycklar finns i `en.json`.```bash
# Basic check
python3 scripts/check_translations.py

# Verbose output
python3 scripts/check_translations.py --verbose

# Auto-fix (adds missing keys to en.json)
python3 scripts/check_translations.py --fix
````

### generate-qa-checklist.mjs

**Statisk analys QA**— skannar Next.js sidfiler för i18n riskmått och genererar en Markdown-rapport.```bash
node scripts/i18n/generate-qa-checklist.mjs

**Checkar:**

- Klassanvändning med fast bredd (spillrisk)

### untranslatable-keys.json

**Fil:**`scripts/i18n/untranslatable-keys.json`

Tillståndslista över nycklar som ska förbli identiska med engelsk källa. Används av `validate_translation.py` för att undvika falskt positiva "oöversatta" varningar.```json
{
"description": "Keys that should remain untranslated...",
"keys": [
"common.model",
"common.oauth",
"health.cpu",
...
]
}

````

**Vad hör hit:**

- Varumärke/produktnamn: "landing.brandName", "common.social-github".
- Tekniska termer/akronymer: "health.cpu", "mcpDashboard.pid", "settings.ai"
- ICU/formatsträngar: `apiManager.modelsCount`, `health.millisecondsShort`
- Platshållarvärden: `providers.openaiBaseUrlPlaceholder`, `cliTools.baseUrlPlaceholder`
- Protokollnamn: `common.http`, `common.oauth`, `providers.oauth2Label`
- Navigeringssektioner: `sidebar.primarySection`, `sidebar.cliSection`

**För att lägga till en nyckel:**Redigera arrayen `nycklar` i `scripts/i18n/untranslatable-keys.json` och kör valideringen igen.## CI Integration

## Best Practices

### When Editing Translations

1.**Redigera alltid `en.json` först**– det är källan till sanningen
2.**Kör `generate-multilang.mjs messages`**för att sprida nya nycklar till alla språk
3.**Granska automatiska översättningar**— Google Translate är en utgångspunkt, inte slutgiltig
4.**Validera innan bekräftelse**— `python3 scripts/validate_translation.py quick -l <lang>`
5.**Uppdatera `untranslatable-keys.json`**om en nyckel ska finnas kvar på engelska### Placeholder Safety

- ICU-platshållare (`{count}`, `{value}`, `{total}`, `{seconds}`) måste bevaras exakt
- Plurala format (`{count, plural, one {# model} other {# models}}`) måste bibehålla struktur
- Validatorn upptäcker platshållarfel automatiskt### Adding New Translation Keys in Code

```tsx
// Use namespaced keys
const t = useTranslations("settings");
t("cacheSettings"); // maps to settings.cacheSettings in JSON

// Run check_translations.py to verify keys exist
python3 scripts/check_translations.py --verbose
````

### RTL Considerations

- Arabiska (`ar`) och hebreiska (`he`) är RTL-språk
- Undvik hårdkodad "vänster"/"höger" CSS - använd "start"/"slut" logiska egenskaper
- Visual QA fångar RTL-layout som inte matchar via `run-visual-qa.mjs`## Known Issues & History

### `in.json` → `hi.json` Fix

Generatorn använde ursprungligen `code: "in"` (utfasad Google Translate-kod) för hindi istället för korrekt ISO 639-1 `hi`. Detta skapade en föräldralös "in.json"-duplicering av "hi.json". Fixat genom att ändra `code: "in"` till `code: "hi"` i `generate-multilang.mjs` och ta bort den föräldralösa filen.### `docs/i18n/README.md` Is Auto-Generated

Filen `docs/i18n/README.md` är helt återskapad av `generate-multilang.mjs docs`. Alla manuella redigeringar kommer att gå förlorade. Använd `docs/I18N.md` (denna fil) för handskriven dokumentation som bör finnas kvar.### External Untranslatable Keys List

Tillståndslistan `untranslatable-keys.json` flyttades från en inline Python-uppsättning i `validate_translation.py` till en extern JSON-fil för enklare underhåll. Validatorn laddar den vid körning.### `generate-multilang.mjs` Hindi Code Fix

Generatorn använde ursprungligen `code: "in"` (utfasad Google Translate-kod) för hindi istället för korrekt ISO 639-1 `hi`. Detta introducerades i upstream commit `952b0b22c` av `diegosouzapw`. Fixat genom att ändra `code: "in"` till `code: "hi"` i `LOCALE_SPECS`-arrayen och ta bort den föräldralösa `in.json`-filen.### `validate_translation.py` Ignored Count Output
