### Source of Truth

-**Strings de UI**: `src/i18n/messages/en.json` (fonte em inglês, ~2800 chaves) -**Arquivos locais**: `src/i18n/messages/{locale}.json` (30 traduções) -**Framework**: `next-intl` com resolução de localidade baseada em cookie -**Config**: `src/i18n/config.ts` — define todos os 30 locais, nomes de idiomas, sinalizadores### Runtime Flow

1. O usuário seleciona o idioma → conjunto de cookies `NEXT_LOCALE`
2. `src/i18n/request.ts` resolve a localidade: cookie → cabeçalho `Accept-Language` → fallback `en`
3. A importação dinâmica carrega `messages/{locale}.json`
4. Os componentes usam `useTranslations("namespace")` e `t("key")`### Supported Locales

### 1. Register the Locale

Edite `src/i18n/config.ts`:```ts
// Add to LOCALES array
"xx",
// Add to LANGUAGES array
{ code: "xx", label: "XX", name: "Language Name", flag: "🏳️" },

````

### 2. Add to Generator

Edite `scripts/i18n/generate-multilang.mjs` — adicione entrada em `LOCALE_SPECS`:```js
{
  code: "xx",
  googleTl: "xx",
  label: "XX",
  flag: "🏳️",
  languageName: "Language Name",
  readmeName: "Language Name",
  docsName: "Language Name",
},
````

### 3. Generate Initial Translation

```bash
node scripts/i18n/generate-multilang.mjs messages
```

Isso cria `src/i18n/messages/xx.json` traduzido automaticamente de `en.json` via Google Translate.### 4. Review & Fix Auto-Translations

As traduções automáticas são um ponto de partida. Revise manualmente para:

- Precisão técnica
- Terminologia apropriada ao contexto
- Tratamento adequado de espaços reservados (`{count}`, `{value}`, etc.)### 5. Validate

```bash
python3 scripts/validate_translation.py quick -l xx
python3 scripts/validate_translation.py diff common -l xx
```

### 6. Generate Translated Documentation

```bash
node scripts/i18n/generate-multilang.mjs docs
```

## Auto-Translation Pipeline

### generate-multilang.mjs (Google Translate)

**Mecanismo principal de tradução automática**— usa a API gratuita do Google Translate para gerar traduções para strings de UI, READMEs e documentação.```bash
node scripts/i18n/generate-multilang.mjs [messages|readme|docs|all]

````

| Modo | O que faz |
| ---------- | ----------------------------------------------------------------------------- |
| `mensagens` | Traduz chaves ausentes em `src/i18n/messages/{locale}.json` de `en.json` |
| `leia-me` | Traduz `README.md` em todas as localidades como `README.{code}.md` na raiz do projeto |
| `docs` | Traduz `DOC_SOURCE_FILES` em `docs/i18n/{locale}/{docName}` |
| `todos` | Executa todos os três modos |

**Recursos:**

-**Proteção de texto**: Mascara blocos de código (` ``` `), código embutido (`` ` ``), links/imagens de markdown (`[text](url)`), tags HTML, tabelas e espaços reservados ICU (`{count}`, `{value}`, `{total}`, etc.) antes da tradução e, em seguida, os restaura
-**Lote fragmentado**: une várias strings com delimitadores `__OMNIROUTE_I18N_SEPARATOR__` para minimizar chamadas de API (máximo de 1.800 caracteres por solicitação)
-**Cache na memória**: evita chamadas de API redundantes para strings repetidas em uma sessão
-**Lógica de nova tentativa**: espera exponencial (até 5 tentativas com 300 ms × atraso de tentativa) para erros 429/5xx
-**Tempo limite**: 20 segundos por solicitação
-**Ignorar existente**: Se o arquivo de destino já existir, ele NÃO será substituído

**Comportamentos importantes:**

- `docs/i18n/README.md` é**regenerado**a cada execução — é um índice gerado automaticamente de todos os documentos
- Arquivos raiz `README.{code}.md` só são criados se não existirem (ignora localidades em `EXISTING_README_CODES`)
- Barras de idiomas (`🌐**Idiomas:**...`) são inseridas/atualizadas automaticamente em todos os documentos traduzidos### i18n_autotranslate.py (LLM-based)

**Tradutor secundário**— usa qualquer API LLM compatível com OpenAI (incluindo o próprio OmniRoute) para traduzir arquivos markdown `docs/i18n/` existentes. Melhor para aprimorar ou retraduzir documentos com melhor qualidade do que o Google Tradutor.```bash
python3 scripts/i18n_autotranslate.py \
  --api-url http://localhost:20128/v1 \
  --api-key sk-your-key \
  --model gpt-4o
````

**Recursos:**

- Verifica arquivos markdown `docs/i18n/` em busca de parágrafos em inglês
- Ignora blocos de código, tabelas e conteúdo já traduzido
- Envia parágrafos para LLM com prompt do sistema de tradução técnica
- Suporta todos os 30 idiomas## Validation & QA

### validate_translation.py

**Validador de tradução**— compara qualquer JSON de localidade com `en.json` e relata problemas.```bash

# Quick check (counts only)

python3 scripts/validate_translation.py quick -l cs

# Output:

# Missing: 0

# Untranslated: 0

# Ignored (UNTRANSLATABLE_KEYS): 236

# Detailed diff by category

python3 scripts/validate_translation.py diff common -l cs
python3 scripts/validate_translation.py diff settings -l cs

# Export to CSV

python3 scripts/validate_translation.py csv -l cs > report.csv

# Export to Markdown

python3 scripts/validate_translation.py md -l cs > report.md

# Full report (default)

python3 scripts/validate_translation.py -l cs

````

**Detecta:**

-**Chaves ausentes**— chaves em `en.json`, mas não no arquivo de localidade
-**Chaves extras**— chaves no arquivo de localidade, mas não em `en.json`
-**Chaves não traduzidas**— chaves em que o valor da localidade é igual à fonte em inglês (excluindo a lista de permissões)
-**Incompatibilidades de marcadores**— marcadores de ICU que não correspondem entre a fonte e a tradução

**Códigos de saída:**
| Código | Significado |
|------|---------|
| 0 | OK |
| 1 | Erro genérico |
| 2 | Strings ausentes (erro grave) |
| 3 | Aviso não traduzido (suave) |

**Ambiente:**Defina `TRANSLATION_LANG=cs` ou use o sinalizador `-l cs`.### check_translations.py

**Verificador de chave de código para JSON**— verifica `src/**/*.tsx` e `src/**/*.ts` em busca de chamadas `useTranslations()` e verifica se todas as chaves referenciadas existem em `en.json`.```bash
# Basic check
python3 scripts/check_translations.py

# Verbose output
python3 scripts/check_translations.py --verbose

# Auto-fix (adds missing keys to en.json)
python3 scripts/check_translations.py --fix
````

### generate-qa-checklist.mjs

**QA de análise estática**— verifica os arquivos da página Next.js em busca de métricas de risco i18n e gera um relatório Markdown.```bash
node scripts/i18n/generate-qa-checklist.mjs

````

**Verificações:**

- Uso de classe de largura fixa (risco de estouro)
- Classes direcionais esquerda/direita (risco RTL)
- Padrões propensos a cortes
- Paridade de localidade (chaves ausentes/extras vs `en.json`)
- Barras seletoras de idioma README em localidades prioritárias (`es`, `fr`, `de`, `ja`, `ar`)

**Saída:**`docs/reports/i18n-qa-checklist-{data}.md`### run-visual-qa.mjs

**Controle de qualidade visual via Playwright**— faz capturas de tela de todas as rotas do painel em vários locais e janelas de visualização e, em seguida, avalia a integridade da página.```bash

**Detecta:**

**Saída:**`docs/reports/i18n-visual-qa-{date}.md` + relatório JSON## Managing Untranslatable Keys

### untranslatable-keys.json

**Arquivo:**`scripts/i18n/untranslatable-keys.json`

Lista de permissões de chaves que devem permanecer idênticas à fonte em inglês. Usado por `validate_translation.py` para evitar avisos falsos positivos "não traduzidos".```json
{
"description": "Keys that should remain untranslated...",
"keys": [
"common.model",
"common.oauth",
"health.cpu",
...
]
}

````

**O que pertence aqui:**

- Nomes de marcas/produtos: `landing.brandName`, `common.social-github`
- Termos/acrônimos técnicos: `health.cpu`, `mcpDashboard.pid`, `settings.ai`
- ICU/strings de formato: `apiManager.modelsCount`, `health.millisecondsShort`
- Valores de espaço reservado: `providers.openaiBaseUrlPlaceholder`, `cliTools.baseUrlPlaceholder`
- Nomes de protocolo: `common.http`, `common.oauth`, `providers.oauth2Label`
- Seções de navegação: `sidebar.primarySection`, `sidebar.cliSection`

**Para adicionar uma chave:**Edite o array `keys` em `scripts/i18n/untranslatable-keys.json` e execute novamente a validação.## CI Integration

O pipeline de CI valida todas as localidades em cada push e PR:

1.**trabalho `i18n-matrix`**— descobre dinamicamente todos os arquivos de localidade (excluindo `en.json`)
2.**`i18n` job**— executa `validate_translation.py quick -l '<lang>'` para cada localidade em paralelo
3.**trabalho `ci-summary`**— agrega resultados em um resumo do painel```yaml

python3 scripts/validate_translation.py quick -l '${{ matrix.lang }}'
````

## Best Practices

### When Editing Translations

1.**Sempre edite `en.json` primeiro**— é a fonte da verdade
2.**Execute `generate-multilang.mjs messages`**para propagar novas chaves para todos os locais
3.**Analise as traduções automáticas**— O Google Tradutor é um ponto de partida, não final
4.**Validar antes de confirmar**— `python3 scripts/validate_translation.py quick -l <lang>`
5.**Atualize `untranslatable-keys.json`**se uma chave permanecer em inglês### Placeholder Safety

- Espaços reservados de UTI (`{count}`, `{value}`, `{total}`, `{seconds}`) devem ser preservados exatamente
- Formatos plurais (`{count, plural, one {# model} other {# models}}`) devem manter a estrutura
- O validador detecta incompatibilidades de espaço reservado automaticamente### Adding New Translation Keys in Code

```tsx
// Use namespaced keys
const t = useTranslations("settings");
t("cacheSettings"); // maps to settings.cacheSettings in JSON

// Run check_translations.py to verify keys exist
python3 scripts/check_translations.py --verbose
````

### RTL Considerations

- Árabe (`ar`) e hebraico (`he`) são localidades RTL
- Evite CSS `left`/`right` codificado — use propriedades lógicas `start`/`end`
- Visual QA detecta incompatibilidades de layout RTL por meio de `run-visual-qa.mjs`## Known Issues & History

### `in.json` → `hi.json` Fix

O gerador originalmente usava `code: "in"` (código obsoleto do Google Translate) para Hindi em vez do ISO 639-1 correto `hi`. Isso criou uma duplicata órfã `in.json` de `hi.json`. Corrigido alterando `code: "in"` para `code: "hi"` em `generate-multilang.mjs` e removendo o arquivo órfão.### `docs/i18n/README.md` Is Auto-Generated

O arquivo `docs/i18n/README.md` é completamente regenerado por `generate-multilang.mjs docs`. Quaisquer edições manuais serão perdidas. Use `docs/I18N.md` (este arquivo) para documentação escrita à mão que deve persistir.### External Untranslatable Keys List

A lista de permissões `untranslatable-keys.json` foi movida de um conjunto Python embutido em `validate_translation.py` para um arquivo JSON externo para facilitar a manutenção. O validador carrega-o em tempo de execução.### `generate-multilang.mjs` Hindi Code Fix
