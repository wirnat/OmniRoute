### Source of Truth

-**यूआई स्ट्रिंग्स**: `src/i18n/messages/en.json` (अंग्रेजी स्रोत, ~2800 कुंजी) -**लोकेल फ़ाइलें**: `src/i18n/messages/{locale}.json` (30 अनुवाद) -**फ्रेमवर्क**: कुकी-आधारित स्थानीय रिज़ॉल्यूशन के साथ `नेक्स्ट-इंटल` -**कॉन्फ़िगर**: `src/i18n/config.ts` - सभी 30 स्थानों, भाषा के नाम, झंडे को परिभाषित करता है### Runtime Flow

1. उपयोगकर्ता भाषा का चयन करता है → `NEXT_LOCALE` कुकी सेट
2. `src/i18n/request.ts` लोकेल को हल करता है: कुकी → `स्वीकार-भाषा` हेडर → फ़ॉलबैक `en`
3. डायनामिक आयात `messages/{locale}.json` को लोड करता है
4. घटक `useTranslations("namespace")` और `t("key")` का उपयोग करते हैं### Supported Locales

### 1. Register the Locale

`src/i18n/config.ts` संपादित करें:```ts
// Add to LOCALES array
"xx",
// Add to LANGUAGES array
{ code: "xx", label: "XX", name: "Language Name", flag: "🏳️" },

````

### 2. Add to Generator

`scripts/i18n/generate-multilang.mjs` संपादित करें - `LOCALE_SPECS` में प्रविष्टि जोड़ें:```js
{
  code: "xx",
  googleTl: "xx",
  label: "XX",
  flag: "🏳️",
  languageName: "Language Name",
  readmeName: "Language Name",
  docsName: "Language Name",
},
````

### 3. Generate Initial Translation

```bash
node scripts/i18n/generate-multilang.mjs messages
```

यह Google अनुवाद के माध्यम से `en.json` से `src/i18n/messages/xx.json` स्वतः अनुवादित बनाता है।### 4. Review & Fix Auto-Translations

ऑटो-अनुवाद एक प्रारंभिक बिंदु है। इसके लिए मैन्युअल रूप से समीक्षा करें:

- तकनीकी सटीकता
- संदर्भ-उपयुक्त शब्दावली
- प्लेसहोल्डर्स का उचित प्रबंधन (`{गिनती}`, `{मान}`, आदि)### 5. Validate

```bash
python3 scripts/validate_translation.py quick -l xx
python3 scripts/validate_translation.py diff common -l xx
```

### 6. Generate Translated Documentation

```bash
node scripts/i18n/generate-multilang.mjs docs
```

## Auto-Translation Pipeline

### generate-multilang.mjs (Google Translate)

**प्राथमिक ऑटो-अनुवाद इंजन**- यूआई स्ट्रिंग्स, रीडमी और दस्तावेज़ीकरण के लिए अनुवाद उत्पन्न करने के लिए Google अनुवाद मुक्त एपीआई का उपयोग करता है।```bash
node scripts/i18n/generate-multilang.mjs [messages|readme|docs|all]

`````

| मोड | यह क्या करता है |
| ---------- | ---------------------------------------------------------------------------------- |
| `संदेश` | `src/i18n/messages/{locale}.json` में गुम कुंजियों का `en.json` से अनुवाद करता है |
| 'रीडमी' | प्रोजेक्ट रूट में `README.md` को सभी स्थानों में `README.{code}.md` के रूप में अनुवादित करता है |
| `दस्तावेज़` | `DOC_SOURCE_FILES` को `docs/i18n/{locale}/{docName}` में अनुवादित करता है |
| `सभी` | सभी तीन मोड चलाता है |

**विशेषताएं:**

-**पाठ सुरक्षा**: अनुवाद से पहले मास्क कोड ब्लॉक (```` `), इनलाइन कोड (`` ``), मार्कडाउन लिंक/छवियां (`[पाठ](यूआरएल)`), एचटीएमएल टैग, टेबल और आईसीयू प्लेसहोल्डर (`{गिनती}`, `{मान}`, `{कुल}`, आदि), फिर उन्हें पुनर्स्थापित करता है
-**चंक्ड बैचिंग**: एपीआई कॉल को कम करने के लिए `__OMNIROUTE_I18N_SEPARATOR__` डिलीमिटर्स के साथ कई स्ट्रिंग्स को जोड़ता है (प्रति अनुरोध अधिकतम 1800 वर्ण)
-**इन-मेमोरी कैश**: एक सत्र के भीतर बार-बार स्ट्रिंग के लिए अनावश्यक एपीआई कॉल से बचें
-**पुन:प्रयास तर्क**: 429/5xx त्रुटियों के लिए घातीय बैकऑफ़ (300ms × प्रयास विलंब के साथ 5 प्रयासों तक)
-**टाइमआउट**: प्रति अनुरोध 20 सेकंड
-**मौजूदा को छोड़ें**: यदि लक्ष्य फ़ाइल पहले से मौजूद है, तो इसे अधिलेखित नहीं किया गया है

**महत्वपूर्ण व्यवहार:**

- `docs/i18n/README.md` प्रत्येक रन पर**पुनर्जीवित**होता है - यह सभी दस्तावेज़ों का एक स्वतः-निर्मित सूचकांक है
- रूट `README.{code}.md` फ़ाइलें केवल तभी बनाई जाती हैं यदि वे मौजूद नहीं हैं (`EXISTING_README_CODES` में स्थान छोड़ देता है)
- भाषा पट्टियाँ (`🌐**भाषाएँ:**...`) सभी अनुवादित दस्तावेज़ों में स्वचालित रूप से डाली/अपडेट की जाती हैं### i18n_autotranslate.py (LLM-based)

**माध्यमिक अनुवादक**- मौजूदा `docs/i18n/` मार्कडाउन फ़ाइलों का अनुवाद करने के लिए किसी भी OpenAI-संगत LLM API (स्वयं ओमनीरूट सहित) का उपयोग करता है। Google अनुवाद की तुलना में बेहतर गुणवत्ता वाले दस्तावेज़ों को चमकाने या पुनः अनुवाद करने के लिए सर्वोत्तम।```bash
python3 scripts/i18n_autotranslate.py \
  --api-url http://localhost:20128/v1 \
  --api-key sk-your-key \
  --model gpt-4o
`````

**विशेषताएं:**

- अंग्रेजी पैराग्राफ के लिए `docs/i18n/` मार्कडाउन फ़ाइलों को स्कैन करता है
- कोड ब्लॉक, टेबल और पहले से अनुवादित सामग्री को छोड़ देता है
- तकनीकी अनुवाद प्रणाली संकेत के साथ एलएलएम को पैराग्राफ भेजता है
- सभी 30 भाषाओं का समर्थन करता है## Validation & QA

### validate_translation.py

**अनुवाद सत्यापनकर्ता**- किसी भी स्थानीय JSON की तुलना `en.json` से करता है और समस्याओं की रिपोर्ट करता है।```bash

# Quick check (counts only)

python3 scripts/validate_translation.py quick -l cs

# Output:

# Missing: 0

# Untranslated: 0

# Ignored (UNTRANSLATABLE_KEYS): 236

# Detailed diff by category

python3 scripts/validate_translation.py diff common -l cs
python3 scripts/validate_translation.py diff settings -l cs

# Export to CSV

python3 scripts/validate_translation.py csv -l cs > report.csv

# Export to Markdown

python3 scripts/validate_translation.py md -l cs > report.md

# Full report (default)

python3 scripts/validate_translation.py -l cs

````

**पता लगाता है:**

-**गुम कुंजियाँ**- `en.json` में कुंजियाँ, लेकिन लोकेल फ़ाइल में नहीं
-**अतिरिक्त कुंजियाँ**- स्थानीय फ़ाइल में कुंजियाँ लेकिन `en.json` में नहीं
-**अनअनुवादित कुंजियाँ**- कुंजियाँ जहाँ स्थानीय मान अंग्रेजी स्रोत के बराबर है (अनुमति सूची को छोड़कर)
-**प्लेसहोल्डर बेमेल**— आईसीयू प्लेसहोल्डर जो स्रोत और अनुवाद के बीच मेल नहीं खाते

**निकास कोड:**
| कोड | मतलब |
|------|---------|
| 0 | ठीक है |
| 1 | सामान्य त्रुटि |
| 2 | गुम तार (कठिन त्रुटि) |
| 3 | अअनुवादित चेतावनी (मुलायम) |

**पर्यावरण:**`TRANSLATION_LANG=cs` सेट करें या `-l cs` ध्वज का उपयोग करें।### check_translations.py

**कोड-टू-JSON कुंजी चेकर**- `useTranslations()` कॉल के लिए `src/**/*.tsx` और `src/**/*.ts` को स्कैन करता है और `en.json` में मौजूद सभी संदर्भित कुंजियों को सत्यापित करता है।```bash
# Basic check
python3 scripts/check_translations.py

# Verbose output
python3 scripts/check_translations.py --verbose

# Auto-fix (adds missing keys to en.json)
python3 scripts/check_translations.py --fix
````

### untranslatable-keys.json

**फ़ाइल:**`स्क्रिप्ट/i18n/untranslatable-keys.json`

कुंजियों की अनुमति सूची जो अंग्रेजी स्रोत के समान रहनी चाहिए। झूठी-सकारात्मक "अअनुवादित" चेतावनियों से बचने के लिए `validate_translation.py` द्वारा उपयोग किया जाता है।```json
{
"description": "Keys that should remain untranslated...",
"keys": [
"common.model",
"common.oauth",
"health.cpu",
...
]
}

````

**यहाँ क्या है:**

- ब्रांड/उत्पाद नाम: `landing.brandName`, `common.social-github`
- तकनीकी शब्द/संक्षिप्त शब्द: `health.cpu`, `mcpDashboard.pid`, `settings.ai`
- आईसीयू/प्रारूप स्ट्रिंग: `apiManager.modelsCount`, `health.मिलीसेकंडशॉर्ट`
- प्लेसहोल्डर मान: `providers.openaiBaseUrlPlacefolder`, `cliTools.baseUrlPlacefolder`
- प्रोटोकॉल नाम: `common.http`, `common.oauth`, `providers.oauth2Label`

## Best Practices

### When Editing Translations

1.**हमेशा `en.json` को पहले संपादित करें**- यह सत्य का स्रोत है
2.**सभी स्थानों पर नई कुंजियाँ प्रसारित करने के लिए `generate-multilang.mjs messages`* चलाएँ
3.**ऑटो-अनुवाद की समीक्षा करें**- Google अनुवाद एक प्रारंभिक बिंदु है, अंतिम नहीं
4.**कमिट करने से पहले सत्यापित करें**- `python3 स्क्रिप्ट/validate_translation.py Quick -l <lang>`
5.**यदि कोई कुंजी अंग्रेजी में रहनी चाहिए तो `untranslatable-keys.json`* को अपडेट करें### Placeholder Safety

- आईसीयू प्लेसहोल्डर्स (`{गिनती}`, `{मान}`, `{कुल}`, `{सेकंड्स}`) को सटीक रूप से संरक्षित किया जाना चाहिए
- बहुवचन प्रारूप (`{गिनती, बहुवचन, एक {# मॉडल} अन्य {# मॉडल}}`) को संरचना बनाए रखनी चाहिए
- सत्यापनकर्ता स्वचालित रूप से प्लेसहोल्डर बेमेल का पता लगाता है### Adding New Translation Keys in Code

```tsx
// Use namespaced keys
const t = useTranslations("settings");
t("cacheSettings"); // maps to settings.cacheSettings in JSON

// Run check_translations.py to verify keys exist
python3 scripts/check_translations.py --verbose
````

### RTL Considerations

- अरबी (`ar`) और हिब्रू (`he`) RTL स्थान हैं
- हार्डकोडेड `लेफ्ट`/`राइट` सीएसएस से बचें - `स्टार्ट`/`एंड` तार्किक गुणों का उपयोग करें
- विज़ुअल QA `run-visual-qa.mjs` के माध्यम से RTL लेआउट बेमेल को पकड़ता है## Known Issues & History
